Configuring the Java Plugin Version for Content Explorer

Content Explorer requires a Java plugin, which must be downloaded to the client when the user browser initiates the Content Explorer applet. The user must be able to connect to a location to download a Java plugin when starting Content Explorer for the first time.

The Java plugin does not work consistently across operating systems and web browsers. Different versions of the plugin exhibit different behavior on various combinations. To help you address this problem, you can specify the version of the Java plugin you want your users to use for specific combinations of operating system and browser.

You can define the type of Versioning to use when specifying the plugin. Options are:

  • Static Versioning

Choose this option if you want to force all user to use exactly the plugin version you specify. If a user has a higher version of the plugin, it will be downgraded to the specified version.

  • Dynamic

Choose this option if you want to allow your users to use higher versions of the plugin than the one you specify.

Use the Content Explorer Tab of the Rhythmyx Server Administrator to specify the Java Plugin versions.

Columns

Client OS Defines the Rhythmyx client operating system for the Java plugin configuration. Options are the following:

  • Any (use this Java plugin with any operating system)
  • Windows NT
  • Windows 2000
  • Windows XP
  • Mac OS X

Browser Defines the browser for the Java plugin configuration. Options are the following:

  • Any (use this Java plugin with any browser)I
  • Internet Explorer 5.0 (IE 5.0)
  • Internet Explorer 5.5 (IE 5.5)
  • Internet Explorer 6.0 (IE 6.0)
  • Netscape 4.79 (NS 4.79)
  • Netscape 7.0 (NS 7.0)
  • Firefox 1.0
  • Safari

Plug-in Version The Java plugin Version for the specification. When you double-click in this field, Rhythmyx displays a drop list showing the known and currently supported versions of the plugin. You can also manually enter a version for the plugin.

Versioning Drop list specifying the type of plugin Versioning used for the Java plugin configuration. Details about these options are described above.

Plugin Location The location of the plugin cab file. When you double-click in this field, Rhythmyx displays a drop list showing the currently known and supported locations for downloading the plugin. You can also manually specify a Rhythmyx location, such as ../rx_resources/Plugin/j2re-1_4_1_03-windows-i586.exe. Be sure the plugin version at this location matches the Plug-in Version you specify. The user must be able to access this location to download and install the Java plugin.
