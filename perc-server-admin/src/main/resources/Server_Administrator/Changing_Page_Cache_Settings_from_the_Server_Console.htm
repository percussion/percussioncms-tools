Changing Page Cache Settings from the Server Console

Rhythmyx provides several cache console commands for managing the page cache from the server console or through the Server Administrator using a remote console command.

Command

Description

flush cache <[keys]>

Flushes the cache based on the provided keys. [keys] is a semi-colon delimited list of keys that you must provide in the correct order. [type] is the page item type, and may be either assembler or resource. You must enter a [type] if you enter [keys].

The keys that you may use to flush a set of assembler items are:

  • Application name (case sensitivity is determined by server setting)
  • ContentId
  • RevisionId
  • VariantId

    You may omit keys, but you must provide a placeholder semi-colon for any that are skipped. If you specify RevisionId, you must also specify ContentId.

    For example, to flush all pages based on ContentId=135 and RevisionId=1:

    flush cache assembler ;135;1;

    The keys that you must use to flush a resource item are:

  • Application name
  • Dataset name

    You may omit either or both keys, but you must provide a placeholder semi-colon for any that are skipped.

    For example, to flush the page based on application name= sys_Compare and Dataset=dataset145:

    flush cache resource sys_compare;dataset145

    To flush the entire cache, omit the [type] and [keys]:

    flush cache

start cache

Enables the cache if it is not already enabled. Does not affect the enabled setting in the server configuration. Enables both assembler and resource caching.

stop cache

Flushes all cache items and stops the cache if it is running. Displays the cache statistics. Does not affect the enabled setting in the server configuration. Flushes both assembler and resource page items in the cache.

restart cache

Stops and restarts the cache. (Clears the statistics and flushes the cache.) Displays the cache statistics. Stops and restarts assembler and resource caching.

dump cache

Displays the following cache statistics to the console:

  • Hit rate (# of hits / total # of requests)
  • Total hits - Total misses
  • Total number of items in the cache
  • Total memory used
  • Total disk space used
  • Average size of an item
  • Disk hit rate (# of hits retrieved from disk/ total # of hits). This information is also displayed by the dump resources command.

    Displays combined statistics for assembler and resource caching.
