Adding a DBMS Table Security Provider

To add a DBMS Table security provider:

  1. On the Security Provider subtab of the Security tab, click the [New] button.
  2. The Server Administrator displays the Select new security provider type dialog. Choose the Back-end table security provider and click the [OK] button.

    The Server Administrator displays the DBMS Table Security Property dialog with the Provider Properties tab selected.

  3. Enter a Provider Name.
  4. Select the Backend Connection tab.
    1. Choose the Datasource you want to use to connect to the database where the authentication data is stored. Options include all Datasources defined in the system.
    2. Choose the Table in which your authentication data is stored.
  5. Select the Authentication tab.
    1. In the User Id column drop list, choose column where user names are stored. Options include all columns in the specified table.
    2. In the Password column drop list, choose the column where passwords are stored. Options include all columns in the table.
    3. If you want to store and query the password in encrypted form, in the Filter Extension Name drop list, choose the password filter to use to encrypt and decrypt the password.
  6. If you want to retrieve data from other columns as user attributes:
    1. Select the Attributes tab.
    2. To add all columns as attributes, click the [Add All] button.
    3. To add a single attribute, click the new button .

      The Server Administrator opens a new line on the tab.

    4. use the following code to associate a user attribute with a column name:

      User/Attribute/attributename=columnname

  7. Click the [OK] button.
