Implementing Single Sign-on

Single Sign-on is a log-in option available when interfacing to Rhythmyx through a J2EE-compliant container. Single Sign-on allows users to access the Rhythmyx server by logging in to their Web application server. If Single Sign-on is not enabled, users must log in to the Web application server and then log in to Rhythmyx again. When using Single Sign-on, users access Rhythmyx browser applications by directing their browser to the Web application server. The Rhythmyx servlet on the Web application server directs Rhythmyx requests, with user credentials, to the Rhythmyx server. When using Single Sign-on and deriving role names from the J2EE server, the role names on the J2EE server must match the names of the Roles in Rhythmyx, including matching case. Thus, you need to account for the names of your roles on your J2EE server when assigning the names to Roles in Rhythmyx. For example, Rhythmyx ships with the default Roles "Admin" and "Editor". If you have roles with the names "Admin" and "Editor" (matching the case in Rhythmyx) on your J2EE server, the members of these Roles on the J2EE server will also have privileges in Rhythmyx.
