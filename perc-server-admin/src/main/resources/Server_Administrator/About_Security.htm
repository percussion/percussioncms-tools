About Security

Rhythmyx can interface to a number of different security provider types to authenticate users, designers, and administrators.  A security provider is a system that:

  • accepts user credentials;

or

  • maintains a directory of user accounts and their credentials.

A security provider may do both.  

When a user provides credentials, Rhythmyx passes the credentials to the security provider.  The security provider then authenticates the user.  Authentication permits a server to identify a user and confirm that the user is who they claim to be.  Security providers perform authentication after a user supplies their credentials.  Credentials identify a user and are usually a log in name and password.  Credentials can also be certificates, or in the most simple form, an IP address.  When the credential is a DNS name, the syntax is *.percussion.com. See Blank and Multiple Security.

Each ACL entry (in the Security tab of the Server Administrator) is associated with a security provider.  If the same ACL entry is used for more than one security provider, each entry can have its own permission level.  For example, a user named "Chris" with Oracle as a security provider could have different permissions than a user named "Chris" who has NT as a security provider.

Administrators define security providers at the server level, but existing security providers can also be assigned to applications.

Rhythmyx authenticates users against security providers in the order that they are specified in the Rhythmyx Server Administrator.

Rhythmyx supports the following security provider types:

  • SPNEGO (for more information, see Single Sign-on Using SPNEGO in help.percussion.com/cmsystem.
  • Directory Connection
  • Web server
  • Backend Table
