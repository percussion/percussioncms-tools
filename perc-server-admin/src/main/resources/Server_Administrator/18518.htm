Server ACL Tab

Use the Security tab's Server ACL sub-tab to set various types of server access for administrators and other users who perform server-level tasks.

Fields on the Server ACL sub-tab:

Entry - The user, group, or Role with the access defined in the Entry's row. By default, two Entries are entered. Default is the Entry for all users; it gives them all types of access except Admin. Admin is the Entry for all users in the Admin Role; it gives them all types of access.

Design Access - Design access enables the user, group or Role in Entry to log in to the Rhythmyx Workbench and modify any design object including Rhythmyx applications. The ACL for each object or application is specified in the Workbench.

Admin Access - This privilege is only given to super-users, such as administrators. It enables them to change the server configuration in the Server Administrator, and to perform other tasks, such as creating a folder under the //Sites or //Folders levels.

Data Access - Not currently used.

Create Access - Enables the user, group, or Role to create XML applications. The user, group, or Role must first have design access before this access has an effect.

Delete Access - Enables the user, group, or role to delete XML applications. The user, group or Role must first have design access before this access has an effect.

If multiple group matches found - For users who appear in more than one Role or group under Entry, the setting chosen in this drop list determines the access level. See Resolving Conflicting Access Settings for explanations of each setting.

Allow detailed messages - Not currently used.

Rhythmyx uses the following rules to determine access levels:

  • User name is explicitly listed in ACL - Use these permissions and no others (for example, do not use those of a Role or group that the user is also listed in).
  • User is listed only in a single group/Role - Use those group or Role permissions only.
  • User is listed in multiple group/Role entries - Use the settings defined in If multiple group matches found field.
  • User isn't listed by means of any group/role or directly - Use Default settings only.
  • User isn't authenticated - Use anonymous settings only. Note that anonymous settings enable users to perform very few tasks. To actually use Rhythmyx, a user must be authenticated.
