ACL Entry Details/Edit ACL Entry Details dialog

Add or edit an ACL entry using the ACL Entry Details or Edit ACL Entry Details dialog.

To access the ACL Entry Details dialog, click the Security tab of the Rhythmyx Server Administrator, then click the Server ACL tab. On the Server ACL tab, click the [New] button.

To access the Edit ACL Entry Details dialog, click the Security tab of the Rhythmyx Server Administrator, then click the Server ACL tab. On the Server ACL tab, select the ACL entry you want to edit and click the [Edit] button.

Field Descriptions

Name - Name of the entry. You can enter a name or choose Default or Anonymous in the drop list:

Default - Permissions that apply to the default entry apply to authenticated users not listed in the ACL.

Anonymous - Applies to all non-authenticated users, which are users who haven't supplied their name or credentials. Also applies to users who are authenticated if the Default entry isn't defined in the ACL.

Type - Type of ACL entry. Choose from the following options:

User - Indicates that entry applies to the user listed in Name.

Group - Indicates that entry applies to all users in the group listed in Name.

Role - Indicates that entry applies to all users belonging to the Role listed in Name.

Access

Allow design access - Enables the user, group or Role in Name to log in to the Rhythmyx Workbench and modify any design object including Rhythmyx applications. The ACL for each object or application is specified in the Workbench.

Allow admin access - This privilege is only given to super-users, such as administrators. It enables them to change the server configuration in the Server Administrator, and to perform other tasks, such as creating a folder under the //Sites or //Folders levels.

Allow data Access - Not currently used.

Can create applications - Enables the user, group, or Role to create XML applications. The user, group, or Role must first have design access before this access has an effect.

Can delete applications - Enables the user, group, or role to delete XML applications. The user, group or Role must first have design access before this access has an effect.
