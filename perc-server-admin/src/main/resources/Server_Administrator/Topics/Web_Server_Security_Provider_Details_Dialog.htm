Web Server Security Provider Details Dialog

Use the Web Server Security Provider Details dialog to create and manage Web server and Web application server security providers.

Field Descriptions

Security Provider Name Required. Name of the Security Provider. The name can include spaces.

Authentication user header name Required. The HTTP header variable Rhythmyx examines to find the authenticated user. This name must match the name specified in the authUserHeaderName parameter in the setup of the Rhythmyx servlet on the Web server. If Rhythmyx cannot locate this value, the user will not be authenticated by this security provider.

User role list header name Required. The http header variable Rhythmyx examines to determine the Roles to which the authenticated user belongs. These Roles are included in the session.. The name must match the name specified in the userRolesHeaderName parameter in the setup of the Rhythmyx servlet on the Web server. The value is case-insensitive and the default value is RxUserRoles. If Rhythmyx cannot locate this value, it queries for the Roles internally on the server.

Role list delimiter Required. Specifies the character or characters used to separate each Role in the list of Roles. The default value is a semicolon (;). For example, Author;Editor.
