Security Providers and Authentication

Security providers perform authentication after a user supplies their credentials. Web designers can choose which security providers they want to use for an ACL. Administrators define the list of possible security providers that you choose from.

Each ACL entry is associated with a security provider. If the same ACL entry name is used for more than one security provider, each entry can have its own permission level. For example, a user named "Bill" with Oracle as a security provider could have different permissions than a user named "Bill" who has NT as a security provider.

You can choose not to specify a security provider type by selecting <any> in the security provider list. Also, you can choose to use <any> type meaning that, once you set up instances of that security provider type, the server can authenticate with any instances of that type. Rhythmyx authenticates users against the security providers in the order that the Administration Applet specifies.

Example:

If you select <any> Database Directory, the server can authenticate the user with any security providers that are of the type Database Directory. You can have several providers of the same type, for example, one for Oracle and one for MS SQL, but both are the type Database Directory.
