Adding a Field Transform

Add a read only rule when you are defining a field in a Content Type or when you are defining a shared field.

For a graphic of the Field Editability editor and definitions of the fields discussed below, see Field Editability Editor.

To add a read only rule:

  1. In the Field Properties Editor, click [Read Only].

    The Field Editability editor opens.

  2. Click in the first available row in the Rule table.
  3. In the Rule Details box, in the Rule type drop list choose a read only rule type.

    Depending on the read only rule type that you choose, fields for entering parameters may appear below the drop list.

  4. Enter values for the parameters. See Default Read Only Rules and their Parameters for a list of the default read only rule types and descriptions of their parameters.
  5. Click [Add] to add the read only rule to the first available row in the Rule table.
  6. Repeat steps 3 through 6 to add additional read only rules for the field. Read only rules execute in the order they are listed in the Rule table.
  7. To change the order in which a read only rule is executed on a field, select the read only rule in the Rule table and click or .
  8. To insert a read only rule between two existing rules in the Rule table:
    1. Select the lower rule and click +.

      An empty row appears between the two existing rules.

    2. Follow steps 3 through 6 to add a rule to in the empty row.
  9. Click [OK].

    The read only rules entered in the Rule table are applied to the field, and the Read Only Rule editor closes. The Field Properties editor reopens.

Now you can:

  • Configure additional properties.
  • Click [Visibility] to create visibility rules.
  • Click [Validation] to create field validation rules.
  • Click [Transform] to create field transformations.
