Defining Fields

For a graphic of the Content Type tab and definitions of the fields discussed below, see Content Type tab.

To define fields of the Content Type:

  1. In the Content Type editor, click the Content Type tab (the first tab).

    The mandatory system fields are already included as part of the Content Type. These fields are:

    • sys_title - system title for the Content Item
    • sys_communityid - ID of Community associated with the Content Item
    • sys_lang - ID of Locale associated with the Content Item
    • sys_corrective - (used for internal processing)
    • sys_workflowid - ID of the Workflow assigned to the Content Item
    • sys_hibernateVersion - (used for internal processing)
  2. To add a system field or a shared field, select the field in the Shared and System Fields selection box and click [>]
    or
    to add multiple shared or system fields, select multiple shared or system fields in the Shared and System Fields selection box and click [>].
  3. To add a local field or field set and define its properties, follow the procedure in Creating Fields and Field Sets.

    If you create a field set, the editor creates a new tab for the field set. Click the tab for each field set that you have created and define the field set.

You have now defined the fields and field sets that make up the Content Type.

Now you can:

  • define any field sets
  • add item transforms, item validations, or pre- and post-processing extensions to the Content Type
  • disable searching for the Content Type
