Object ACL Dialog

The Object ACL dialog allows you to set the levels of access that Roles or Communities have to an object listed in Rhythmyx workbench views.

To access the Object ACL dialog:

  • In any view that displays the object (but not a reference to the object) right-click on the object and choose Security.

Field Definitions

ACL Entries - Roles, Communities, or users in the ACL. There are two special entries:

Default - Defines permissions for a user not represented by another entry in the list.

AnyCommunity - Defines permissions for a Community that is not listed (for objects that are associated with Communities).

[New Entry] - Pressing this button opens the Add ACL Entry dialog for adding an entry to the ACL Entries list.

[Delete] - Pressing this button deletes a selected entry in the ACL Entries list. You cannot delete Default or AnyCommunity.

Design Access - Permissions for working on this object in the Rhythmyx Workbench. Users are given the maximum access of all checked permissions.

Read - Permission to view the settings for this object in the Rhythmyx Workbench.

Update - Permission to change the settings for this object in the Rhythmyx Workbench.

Delete - Permission to delete this object in the Rhythmyx Workbench.

Modify ACL - - Permission to modify the permissions for this object.

Runtime Access - This field only appears for objects that can be associated with Communities. These objects are Content Types, Display Formats, Menus, Menu Entries, Searches, Sites, Templates, XSL Variants, Views, and Workflows. Runtime access gives access to this object in Content Explorer.

Read - Permission to have access to this object in Content Explorer.
