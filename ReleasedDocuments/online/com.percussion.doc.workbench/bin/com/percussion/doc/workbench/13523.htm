Fields and Field Sets Editor

Using the Fields and Field Sets Editor you can:

  • Create new fields and field sets.
  • Configure additional search and display properties
  • Edit fields

To access the Fields and Field Sets Editor:

  • To edit Content Type fields, open the Content Type editor and click the Content Type tab or click any field set tabs. On these tabs, the editor includes buttons for changing the order of fields. On the Content Type tab, the editor also includes a list of shared and system fields.
  • To view system fields, in Content Design View, right-click on the Content Types Global Configuration node and choose Open. This version of the editor does not allow deletion of fields or addition of field sets.
  • To edit shared fields, In Content Design View, right-click on a shared field object and choose Open.

Field Definitions

Fields - Fields or field sets in the Content Type, shared field file, or system field file. Add local or shared fields by entering a name in a cell under Name.

Name - Required. The system name of the field.

Label - The display name for the field.

Control - Required. The mechanism for displaying the field in the Content Editor, such as drop-list or text box. Rhythmyx supplies a set of default controls. If you specify a sys_table control in a Content Type, you are creating a child field set, and a new tab for the child field set appears at the bottom of the editor.

Field Properties - These properties define a field or field set selected in the Fields table.

Data type - Required. The type of data that the field stores, such as text or binary. Default is data type associated with the Control. See Default Content Editor Controls for the default data type associated with each control. This field cannot be edited in the System field editor.

Storage size - Required. The storage size of the field in the backend table. Default is Storage size associated with the Control. See Default Content Editor Controls for the default storage size associated with each control. This field cannot be edited in the System field editor.

Default value - The default value of the field, if any. Click [...] to set the value using a user-defined function or the Value Selector.

Enable searching for this field - Allows search mechanism to search the contents of this field. Default is checked.

Display:

Mnemonic - The key used in the ALT + key sequence for moving the cursor to this field; drop list includes all unused letters for fields in the Content Type.

Show in preview - Specifies whether or not to display this field in a preview of a Content Item. Default is checked.

[All Properties] - Opens the Field Properties Editor.

[Validation] - Opens the Field Validation Editor.

Field Set - This section only appears on the Shared Field editor. It allows users to add another field set to the shared field file.

Field set name - Name of additional field set. A tab for the field set appears when you click [Add Field Set].

The fields have the following definitions:

Mnemonic - The key used in the ALT + key sequence for moving the cursor to this field; drop list includes all unused letters for fields in the Content Type.

Enable searching for this field set - Allows search mechanism to search the contents of the fields in this field set. Default is checked.

Allow user to order entries - Lets users change the order of fields in the field set. Default is checked.

Validation on Workflow Transition:

Occurrence: A value that tells the editor how to validate the child field set during a Workflow Transition. Possible values are optional, required, and specified count. If the value is required, the editor checks to see if at least one entry for the field set is entered and returns an error message if it is not. If the value is specified count, the editor checks to see if at least the number of entries specified in Count are entered and returns an error message if they are not.

Count: Enabled if the value of Occurrence is specified count. The minimum number of entries for the child field set. During a Workflow Transition, if the value of Occurrence is specified count, the editor checks to see if at least the number of entries specified in Count are entered in the child field set and returns an error message if they are not.
