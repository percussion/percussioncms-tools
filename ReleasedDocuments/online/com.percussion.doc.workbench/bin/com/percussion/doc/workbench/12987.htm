References to Objects in Views

In some cases, tree structures in Rhythmyx views list references to objects, rather than the objects themselves. An object is referenced if it appears below another object, but it is not a child of the object. Instead, it is associated with the object.

For example, Slots are not defined as children of Templates, but may be associated with Templates, so references to associated Slots appear under Templates. The actual Slots appear in their own Slots category. In contrast, Keyword Choices are defined in connection with a specific Keyword and are therefore children of that Keyword. The actual Keyword Choice object appears under the Keyword. All references to objects are designated by [xx graphic of icon]. To delete the object from the system, you must delete the actual instance of the object; when you remove the reference, you only remove the association to the parent object.
