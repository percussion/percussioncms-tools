Adding Extensions and Disabling Searching

When you initially access the Properties tab of the Content Type editor, you can:

  • Add item input and output transforms, item validations, and pre- or post-processing extensions to the Content Type. For descriptions of these processes, see Content Type Editor - Properties Tab.
  • Disable searching of the Content Type.
  • Edit fields that you have entered in the New Content Type Wizard. For information about editing these fields, see Editing a Content Type.

By default, Enable searching for this Content Type is checked in the Properties tab. If you do not want this Content Type included when users perform searches, uncheck Enable searching for this Content Type.

The Properties tab includes an Item Transforms and Validations box for setting up item input transforms, item output transforms, item validations, pre-processing extensions, and post-processing extensions.

To set up an item transform, validation, or pre- or post-processing extension:

  1. Open the Content Type editor and click the Properties tab.
  2. In the Item Transforms and Validations box, click the Input Transforms tab, the Output Transforms tab, the Validations tab, the Pre-Processing tab, or the Post-Processing tab.
  3. In the Extension table, Click in the first available row.

    A list of extensions opens.

    [graphic]

  4. Choose an extension.

    If the extension includes parameters, the Extension Parameters dialog opens.

  5. Add or modify parameters for the extension.
  6. The extension appears in the Extensions table. A drop list arrow appears beside the extension name to let you change the extension. If the extension has parameters, ellipses [...] also appear beside it for re-opening the Extensions Parameters dialog.
  7. If you are setting up an item validation, in Maximum Number of Errors Before Stopping, enter the number of errors that validation can find in the Content Type before it stops validating and returns error messages.
  8. If you are setting up an item input or output transform or validation, a C column appears beside the extension. Click in the C column if you want to enter a condition for running the extension.

    You can set up as many item input transforms, item output transforms, item validations, pre-processing extensions, or post-processing extensions as your Content Type requires.

    Note: In versions of Rhythmyx prior to 6.0, item transforms were referred to as item translations.

Now you can:

  • Save the Content Type
  • Create templates for the Content Type
  • Use the Content Type in Content Explorer
