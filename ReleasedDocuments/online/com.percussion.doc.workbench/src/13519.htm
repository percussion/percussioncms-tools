Maintaining Content Type Fields

Rhythmyx Content Types include three types of fields:

  • System - System fields are supplied by Rhythmyx and can appear in all Content Types. You can edit but not add or delete system fields. Do not override validation rules for system fields used in Content Types because an error may occur when you try to save a Content Type that uses the field.
  • Shared - Shared fields are user-defined. You can include them in multiple Content Types. Shared fields are always defined as field sets in shared field files. They appear as objects under the Shared Fields folder in the Content Design View. Do not override validation rules for shared fields used in Content Types because an error may occur when you try to save a Content Type that uses the field.
  • Local - Local fields are user-defined for a specific Content Type. You must open the Content Type object to view its local fields.

The editor for creating and editing fields is nearly identical for System, Shared, and Local fields. In the Field Editor topic, we display the Shared field version of the editor and explain or cross reference any differences that appear in the versions for Local and System fields.

  • Use the Fields and Field Sets Editor to create new Shared and Local fields.
  • Use the Field Properties Editor to configure additional search and display properties of fields.

You can:

  • Create new fields and field sets;
  • Configure additional search and display properties;
  • Edit a field;
  • Delete shared field sets or
  • Open the Content Editor system definition file in a text editor.
