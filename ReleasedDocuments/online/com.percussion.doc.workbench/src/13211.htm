Maintaining XML Server Objects

This section of the Help contains topics describing objects that existed in Rhythmyx prior to Version 6.0.

These objects are required for backward compatibility. In most cases a new object or implementation has superseded these objects, but the objects continue to function so upgraded systems can work.

You can continue to use these objects, but in future releases they will be phased out. When planning future development, you should plan to use the new standard objects introduced with Rhythmyx Version 6.0 rather than older objects. In addition, you should begin to plan re-implementation of your system so it will continue to function when the pre-Version 6.0 objects are phased out.
