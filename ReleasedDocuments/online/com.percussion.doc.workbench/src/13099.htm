CMS Navigator - Content Design

The Content Design view of CMS Navigator displays Content objects and the sub-categories and objects beneath them.

The main objects in the CMS Navigator Content view are Content Types, Keywords, Shared Fields, Localization, and Content Types Global Configuration. Below, descriptions of these objects and their children follow the graphical representation of the Content view.

Screenshot of Content Design View

Content Types Folder

The Content Types folder includes folders representing categories of Content Types and any Content Types that are not inserted in folders. The Content Type folders are:

  • Navigation folder - Provided by default; displays Content Types used to create navigation Content Items (such as Navigation Bars). You cannot add Content Types to this folder; they must be defined in the navigation.properties file.
  • user-defined folders - Any folders that users create; contain user-defined categories of Content Types. In our example, CustomerSite is a user-defined category.

Each Content Type can include sub folders for:

  • Allowed Workflows - includes the names of Workflows that can be assigned to Content Items of the Content Type. To associate a Workflow with the Content Type, check it in the Content Type editor, or drag the Workflow from the Workflows folder in System Design view and drop it on Allowed Workflows folder for the Content Type.
  • Allowed Templates and XSL Variants - includes references to Templates and XSL Variants that are allowed for the Content Type. To add a reference to a Template, drag the template from the Templates folder in Assembly Design view and drop it on the Allowed Templates and XSL Variants folder for the Content Type.

Keywords Folder

The Keywords folder includes nodes for keywords and sub nodes below the keywords for keyword choices.

Shared Fields Folder

The Shared Fields folder includes nodes for each shared field object and sub nodes below the shared field objects for shared field sets.

Localization Folder

The Localization folder includes sub-folder for Locales and Translation Settings. The Locales node includes sub folders for all of the Locales defined in the system. Opening the Translation Settings node displays the editor where all translation settings are defined.

Content Types Global Configuration Folder

The Content Types Global Configuration node represents the system field definition file for Content Editors. You can open the Content Type Global Configuration node either in the Field and Field Set editor or a text editor to view or edit System Fields and their properties. Note that in most systems this file is never edited, and that only advanced users should modify it.
