Maintaining CMS Files

Rhythmyx resources are included under the CMS Files folder in System Design view. Resources are the files that Rhythmyx uses for running the Rhythmyx application, publishing its default Web Site, and running all of the other applications that it uses.

The CMS Files folder in System Design view contains, by default, the sys_resources and rx_resources applications. By default, these applications include files that help build the Rhythmyx interface such as files required for running specialized controls and images used in the Content Explorer interface. The sys_resources folder includes system files and components that are generally not changed. The rx_resources folder includes modified copies of system files and user-added files for building the Rhythmyx interface or publishing the default Web Site. Files used for building the default Web Site include global templates, banners, icons, and other Web graphics.

When users add empty applications to your system, they appear in the CMS Files folder, and all of the files they contain are available for viewing and editing.

The files in the sys_resources folder are generally not meant to be edited because they are overwritten on upgrade.

You can:

  • Add an application to the CMS Files section.
  • Edit or view a file in an application.
  • Delete an application.
