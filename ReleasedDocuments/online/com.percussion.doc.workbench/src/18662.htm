Creating Fields and Field Sets

To create a field or field set, complete the Fields and Field Sets editor.

For a graphic of the editor, instructions for accessing it, and definitions of the fields discussed below, see Field and Field Set Editor.

To create a field or field set:

  1. In the Fields and Field Sets table click the Name cell labelled Click to add and type in the name for the field or child field set. It is best practice to enter the field or child field set Name in lower case. (NOTE: Field names that use an initial lower-case letter followed by an upper-case letter are not valid in Rhythmyx; in other words, aBcd would not be a valid field name, but abcd or even ABCD would be valid.)
  2. The Label (display name) for each field is filled in with the value you enter for Name. If you want to display a different name for the field, type in a new Label.
  3. To add a control for a field, click the down arrow in the Control cell and choose any of the controls in the drop list except sys_table.

    If you choose an array control (sys_CheckBoxGroup, sys_CheckBoxTree, sys_DropDownMultiple), after you save your edits, you can only change the control to another array control. If you choose a non-array control (any other control except sys_table), after you save your edits, you can only change the control to another non-array control.

    To add a child field set, click the down arrow in the Control cell and choose sys_table in the drop list. This control is not available if the Fields and Field Sets table you are editing represents a shared field set.

    A child field set can include any number of sub-fields. For example, a locations field set could include sub-fields for city, address, state, and contact. A user could enter any number of locations and specify a city, address, state, and contact for each one.

    The editor adds a field set tab for each child field set that you enter. The tab includes a Fields table for entering the fields in the child field set in the same way you enter fields in any Fields and Field Sets table.

  4. To add properties to a field, select the field in the Field and Field Sets table, and fill in the properties that appear in the Field Properties section at the bottom of the tab.
    1. By default, the Data Type is determined by the Control selected. Optionally, choose another data type from the Data Type drop list.
    2. By default, the Storage Size is determined by the Control selected. Optionally, choose another Storage Size from the Storage Size drop list.
    3. Optionally, enter a Default Value for the field. Either enter a literal value or click [...] and choose a user-defined function or an option in the Value Selector for assigning a default value.
    4. By default, Enable searching for this field is checked. If you do not want the content in this field included in searches, uncheck Enable searching for this field.
    5. If you want to require that a value be entered in this field, check Required.
    6. Optionally, in the Mnemonic drop list, choose a key that can be used with the sequence ALT + key to move the cursor to the field.
    7. By default, Show in preview is checked. If you do not want the field to be visible when users preview items of the Content Type in Content Explorer, uncheck the Show in preview check box.
    8. Click [All Properties] to set additional properties.
    9. Click [Validation] to set field validation rules.
  5. To add properties to a child field set, select the child field set in the Field and Field Sets table, and fill in the fields that appear under Field Properties section at the bottom of the tab.
    1. Optionally, in the Mnemonic drop list, choose a key that can be used with the sequence ALT + key to move the cursor to the field.
    2. By default, Enable Searching for this Field Set is checked. If you do not want the content in this field set included in searches, uncheck Enable Searching for this Field Set.
    3. By default, Allow User to Order Entries is checked. If you do not want to give Content Editor users the ability to change the order of fields in the field set, uncheck Allow User to Order Entries.
    4. Choose a value in Occurrence. Possible values are:
      • optional - Prior to a Workflow Transition, no validation is performed.
      • required - Prior to a Workflow Transition, the editor checks that at least one entry is entered in the child field set.
      • specified count - Prior to a Workflow Transition, the editor checks that at least the number of entries specified in Count is entered in the child field set. If you enter specified count, the Count field is enabled.
    5. If you have chosen specified count in Occurrence, choose a value in the Count up-down control.
  6. If you are entering a shared field set, if you want to add an additional field set, in the Field Set section enter a Field Set Name and click [Add field set].

    The editor adds a field set tab for the new field set. The tab includes another Fields and Field Sets editor (a Fields table) for creating fields in the field set. Use the instructions in this topic to fill in the fields on the tab.

Now you can:

  • Save your edits and close the editor.
  • Choose a field and click [All Properties] to add Additional Properties.
  • Choose a field and click [Validation] to add field validations.
