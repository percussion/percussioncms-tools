Results Pager

A Result Pager returns data from the query in increments. It defines how many pages of data to return and how many rows to return on each page.

When you double-click on the Results Pager icon, the Workbench displays the Result Pager Properties dialog.

Field Descriptions

Max rows per page Specifies the maximum number of rows of data to include on each page returned. The default value, -1, returns all rows on a single page.

Max pages Specifies the maximum number of pages of data to return. The default value, -1, returns all pages. The combination of values in the Max rows per page and the Max pages fields can limit the number of results returned. For example, assume that your query returned 110 results; assume also that the value of the Max rows per page field is 20, while the value in the Max pages field is 5. You would receive only the first 100 results (5 pages each with 20 rows of data). The remaining ten rows would not be processed.

Max displayed page links Specifies the maximum number of of links to other pages that can be included in the results on a single page. Default value is 10.

Sorting table

This table defines how the output page data will be sorted. Click the new button to add a new column.

Column Specifies the column used for sorting pages. You can add multiple rows to the Sorting criteria. The results are sorted based on the order in which the rows are specified in the Sorting criteria from top to bottom; once the results are sorted according to the first column specified, groups of the same result are sorted according to the second column, and so on. To change to order of the sort, select a column and move it up or down to the correct position in the order.

Ascending Checkbox. If checked, the data will be sorted in ascending order. Otherwise, the data is sorted in descending order.
