Configuring Additional Properties

To enter additional properties in a field defined in the Fields and Field Sets Editor or to access the editors for setting field validation, field visibility rules, and field transforms, open the Field Properties editor. The Field Properties editor is not available for a child field set selected in a Content Type editor's Content Type tab, but it is available for the fields in a child field set displayed on a Content Type editor's <field set> tab.

For a graphic of the Field Properties editor and definitions of the fields discussed below, see Field Properties Editor.

  1. To add additional properties to a field select it in the Field and Field Set editor and click [All Properties].

    The Field Properties editor opens. The Field Name, Data Type, Storage Size, Default Value, Mime type mode, Mime type value, Label, Mnemonic, Control, Show in Preview and Allow this field to be Searched fields are filled in with values from the previous screen. If you are editing a field in a child field set, the editor enables the Show in Summary check box. Optionally, enter any of the following fields:

    1. Error Label is automatically filled with the same value as Label. The system concatenates error messages for the field to the Error Label value. If you want error messages concatenated with a different value, change the value in Error Label.
    2. For a field in a child field set, leave Show in Summary checked to display a field name and its values in the main Content Editor when users open it. Otherwise, users must click [Edit Table] to view the field as part of the entire child field set in a separate window.
    3. The Show 'Clear Field' Checkbox in Binary Control field is automatically enabled if the Data Type of the field is binary. If you want the user to see a checkbox for clearing the data in the backend table column before inserting the data entered in this field, check the check box. If you do not make this field available, users may fail to overwrite data in the backend table with a null or blank value.
    4. If Allow this field to be searched is checked, by default, Include in full text multi-field query is also checked. The function of the Include in full text multi-field query option is to include the field in user searches by default; if you uncheck it, you could give the user the option of choosing the field for searching when creating a content item. If Allow this field to be searched is not checked and you check this option, Allow this field to be searched is automatically checked.
    5. To add Read Only rules for the field, click [Read Only].
    6. To add Visibility rules for the field, click [Visibility].
    7. To add Validation rules for the field, click [Validation].
    8. To add input or output Transforms for the field, click [Transforms].
    9. Click [OK] to save the field properties entered in this editor and return to the Field and Field Sets editor.

Now you can:

  • Continue to add fields in the Field and Field Sets editor
  • Define Read Only rules
  • Define field Visibility rules
  • Define field Validation rules
  • Define field Transforms
