Deleting a Shared Field File or Shared Field Set

You can only delete a shared field file from the Workbench if no Content Types use any of the shared fields that will be deleted; otherwise the system displays an error when you confirm your deletion.

To delete a shared field definition file:

  1. In the Shared Fields node of Content Design view, right-click on the shared field object and choose Delete.

    A prompt asks you to confirm that you want to delete the field set.

  2. Click [OK].

    The shared field object and all of its shared field sets are deleted.

    Note: If any of your Content Types use any of the shared fields that will be deleted, the system displays an error message and you cannot proceed.

You can only delete a shared field set from the Workbench if no Content Types use fields from the shared field set; otherwise, the system displays an error message when you attempt to save the change to the shared field file, and the change is not saved.

To delete a shared field set:

  1. Open the shared field definition file in its editor.
  2. Click the lower tab of the shared field set that you want to delete.
  3. Right-click on the editor (in a location other than the fields table or a label tab).

    The editor displays a [Delete Field Set] button.

  4. Click the [Delete Field Set] button.

    A prompt asks you to confirm that you want to delete the field set.

  5. Click [Yes].

    The shared field set is deleted.

  6. Save the changed shared field file.

    The file is saved. You must restart your server before your system reflects your changes.

    Note that if any of your Content Types use the fields in the shared field set that you are attempting to delete, the system will not save the changes and your shared field set is not deleted.
