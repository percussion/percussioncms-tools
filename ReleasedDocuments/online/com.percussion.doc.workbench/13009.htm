Editing a Content Type

When you edit a Content Type, you can modify any of the information accessible from the Content Type editor and the editors it links to.

To edit fields, field sets, and field properties of a Content Type:

  1. In Content Design View, right-click on the Content Type and choose Open

    OR

    Double-click on the Content Type.

    The Content Type editor opens with data filled in for the Content Type.

  2. To edit Content Type fields, click the Content Type tab. To edit field set fields, click a <field set> tab. See Editing Fields and Field Sets for steps on editing the fields.
  3. To edit Properties of the Content Type such as assigned Workflows and item transforms and validations, click the Properties tab.
    1. You can enter a new Label or Description, or check/uncheck the Enable searching for this Content Type check box.
    2. Check or uncheck any Allowed workflows. If you uncheck the Default workflow, the Default workflow changes to the first checked Workflow.
    3. You can change the Default workflow; it the new Default workflow is not checked under Allowed workflows, it is automatically checked.
    4. To edit Item Input Transforms, Output Transforms, Validation, Pre-processing extensions or Post-processing extensions, click the appropriate tab.
      • To change the order in which an Item Input Transform, Item Output Transform or Item Validation is executed, on the appropriate Item tab select the extension and click or .
      • To delete an extension, select the extension and click .
      • To insert a new extension at the bottom of the table, click [...] beside Select an Extension. To insert a new extension within the existing table, click the extension that you want to appear below it and click . A new row is inserted into the table. Click [...] beside Select an Extension to insert a new extension in the blank row. Choose a new extension and set any parameters for the item input transform, item output transform, item validation, pre- or post-processing extension.
      • For validations, you can change Maximum number of errors before stopping.
