Creating the Content Type

To enter the minimum amount of information to create a Content Type that appears in Content Design View, complete the New Content Type wizard.

For a graphic of the wizard and definitions of the fields discussed below, see New Content Type Wizard.

To create a Content Type:

  1. Open Content Design View.
  2. To create the Content Type in a sub-folder right-click on the Content Type folder in which you want to create the Content Type and choose New > Content Type.

    OR

    To create the Content Type in the main Content Type folder, in the Menu bar, choose File > New > Content Type.

    The New Content Type wizard opens to the first page.

  3. In Content Type name, enter an internal name for the Content Type.

    The Content Type name is automatically entered in the Label field.

  4. Optionally, in Label, change the display name value.
  5. Optionally, in Description, enter a description of the Content Type.
  6. Optionally, choose Communities that have access to the Content Type. If Communities are enabled in your system and you do not choose any Communities, no users will have access to the Content Type.
    1. If you do not want one of the Communities under Visible to these Communities to have access to the Content Type select the Community and click [<] to move it to Available Communities

    or

    Click [<<] to move all Communities under Visible to these Communities to Available Communities.

    1. If you want to reduce the Communities listed under Available Communities, in Filter, enter a pattern that each Community name must match.
  7. Click [Next].

    The Workflows dialog of the Content Type wizard opens.

  8. Choose Workflows that can be associated with the Content Type. At least one Workflow must be listed under Allowed workflows.
    1. If you want to reduce the Workflows listed under Available workflows, in Filter, enter a pattern that each Workflow name must match.
    2. Select Workflows that can be associated with the Content Type under Available Workflows and click [>] to move them to Allowed Workflows

    or

    Click [>>] to move all Available Workflows to Allowed Workflows.

  9. Initially, the Default Workflow is blank. The drop list includes all Available Workflows. Choose a value in the drop list. If the value chosen is not listed under Allowed Workflows, it is automatically moved from Available Workflows to Allowed Workflows. You cannot continue if a Default Workflow is not entered.
  10. Click [Finish].

The Content Type is created and appears in the Content Design View. It only includes mandatory system fields. The Content Type wizard closes and the Content Type editor opens.

Now you can:

  • define the fields as well as optional properties in the tabs of the Content Type editor.
  • close the Content Type editor and fill in the information at a later time.
