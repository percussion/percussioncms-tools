CMS Navigator - System Design

The System Design view of CMS Navigator displays System objects. The files and objects in this view are not frequently modified.

The main objects in the CMS Navigator System Design view are CMS Files, Extensions, Configurations, Relationship Types, Sites, and Workflows. Below, descriptions of these objects and their children follow the graphic of the System Design view.

Screenshot of System Design View

CMS Files Folder

The CMS Files folder includes Rhythmyx's default sys_resources and rx_resources applications, and any empty Rhythmyx application folders that users add to your system. When expanded, each folder displays the files that it stores.

  • sys_resources - Provided by default; contains files that are needed for various parts of the system to work properly, such as XSL stylesheets or JavaScripts. When you upgrade the server (or run the Installer for any reason) these files are overwritten. By default, these files are read-only, but you can allow editing.
  • rx_resources - Provided by default; contains files that implementers can modify after installation. The Installer never modifies them on an upgrade. In most cases, implementers use these files to modify some aspect of server behavior. Users can edit these files.
  • <user applications> - Folders for empty user-added applications.

Extensions Folder

The Extensions folder includes all extension files (generally Java plugins) in the system. You can view or edit these files, if necessary. Under the Extensions folder, the files are categorized by functional category and any user-defined folders created within the category.

The categories of extensions that Rhythmyx supplies are the following:

  • Assembly Location Generators - Extensions that create the locations where Content Items are published or viewed on a Web Site. For more information, see the Publishing section of the Rhythmyx Implementation Guide.
  • Content List Generators - Extensions used to generate lists of Content Items to publish.
  • Java Script - Extensions written in JavaScript.
  • JEXL Expressions - Extensions written in Java Expression Language for processing Rhythmyx Templates. For more information see Maintaining Templates.
  • Password Filters - Extensions for encrypting passwords.
  • Relationship Effects - Rhythmyx extensions available only for Relationship processing. Use Effects to provide processing for Relationships after they have been created. For more information, see the document Implementing the Relationship Engine.
  • Request Pre-processors and Result Document Processors - Function at the resource level. They take multiple inputs and output a document.  These extensions may be processed before or after processing the document. Request Pre-processors are methods of the IPSRequestPreProcessor class. They process the document when the server receives it, before the resource performs any processing. Result Document Processors are methods of the IPSResultDocumentProcessor class.  They process the document after the resource processes it, but before it is returned to the user.
  • Slot Content Finders - Extensions that locate the Content Items inserted into Slots. For more information see the Creating Slots and Templates chapter of the Rhythmyx Implementation Guide.
  • Template Expanders - Extensions that specify the templates that can be used to format a Content Item.
  • UDF Processors - Extensions that function at the item (field) level.  You can map a UDF to a field to generate the field's value. UDFs take one or more inputs and output a single field.  UDFs can be created as Java classes or as scripts in a scripting language such as JavaScript or Perl.  A handler must exist for the scripting language.  A JavaScript handler is included in Rhythmyx.
  • Workflow Actions - A Workflow Action is an automatic action that can be added to a Transition. Rhythmyx executes the Workflow Action when Transitioning a content item from one State to another. For more information, see the document Maintaining Workflows in Rhythmyx.

Configurations Folder

The Configurations folder includes nodes representing configuration files. Each of these files must be edited manually in a text editor.

  • Logging - The file that contains the default settings for logging (log4j.properties).
  • Navigation - The file that sets default properties for navons, navimages, and navtrees (navigation.properties).
  • Server Page Tags - File for modifying XSL tags that XSpLit cannot handle (serverPageTags.xml).
  • System Velocity Macros - The file of system Velocity macros. Do not edit this file.
  • Thumbnail URL - The file for mapping thumbnail Variants to Content Types for use with Display formats (addThumbnailURL.properties).
  • Tidy - The file that sets the default properties of HTMTidy that Rhythmyx uses for XSpLit (tidy.properties).
  • User Velocity Macros - A file for including Velocity macros that override system Velocity macros.
  • Workflow - The file that sets default properties for processing Workflow actions (rxworkflow.properties).

Relationship Types Folder

The Relationship Types folder lists all Relationship Types on the system. It contains two categories of Relationships:

  • System - Includes Relationships supplied by Rhythmyx.
  • User - Includes user-defined Relationships.

Sites Folder

The Sites folder lists all Sites registered with the system. Register Sites on Content Explorer's Publishing tab.

Workflow Folder

The Workflow folder lists all Workflows configured for the system. Each Workflow includes an Allowed Content Types folder. Configure Workflows on Content Explorer's Workflow tab.
