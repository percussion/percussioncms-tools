Rule Details box

Parameter definitions

Required - A value for the field must be entered.

 

 

None

Number Range - Numeric value must fall within the specified number range

Together, the parameters define a range. For example:

0 < [selected field] < 999999

In the upper and lower drop lists, enter numbers or choose variables that store numbers.

Date Range - The value in the field must fall within the specified date range.

Together, the parameters define a date range. For example:

05012005 <= [selected field] <=05302005

In the upper and lower drop lists, enter dates, choose the today variable, or choose other variables that store dates.

String Length - The length of the string value in the field must fall within the valid length range.

Minimum length - minimum valid length of string. Enter a number or choose a variable that stores a length value.

Maximum length - maximum valid length of string. Enter a number or choose a variable that stores a length value.

Regular Expression - The value in the field must conform to the format of the regular expression.

Choose a Predefined format or enter a regular expression defining a format in Regular expression.

Predefined - values are:

no whitespace

phone

email

url

isNumber

isAlpha

Regular expression - Enter a regular expression.

JEXL Expression -

Enter a JEXL expression that defines that validation rule for the field.

Conditionals - The value of the field must conform to the manually entered conditions.

Variable - Click on [...] to open the Value Selector and choose a value.

Op - Choose a conditional operator in the drop list.

Value - Click on [...] to open the Value Selector and choose a value.

Boolean - Enter AND or OR to show the relationship to the condition in the next row.

Extension - The extension evaluates the field for validity.

Name - Extension that will validate value of field.

[...] - Opens Extensions Parameters dialog
