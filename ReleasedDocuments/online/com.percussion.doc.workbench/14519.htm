Snippet Drawer

The Snippet Drawer contains a set of default Velocity macros used when formatting Velocity Templates. By default, the Snippet drawer is located on the lower left corner of the Rhythmyx Workbench interface, but, like other interface elements, you can move it to another location that you find more convenient.

Screenshot of Snippet Drawer

The Snippet drawer contains three sections:

  • Rx Field Macros

    This section lists the default macros used to add fields to a Velocity Template. The following field macros are available:

    • field [#field(fieldname)]

      This macro is used to add most fields to the Template. Fields added using this macro are displayed with the icons used for Active Assembly. The fieldname parameter specifies the field to be added to the Template.

    • displayfield [#displayfield(fieldname)]

      This macro is used to add fields to the Template if the fields should not be available to be edited, such as inserting the Display Title to the <title> tag of the HTML header. Fields added using this macro are displayed without the icons used for Active Assembly. The fieldname parameter specifies the field to be added to the Template.

    • fieldLink [#fieldLink(fieldname,$pagelink)

      This macro is used to add fields to the Template when the value of the field will be the content of an anchor tag (<a>) and you want users to be able to follow the link rather than opening the field to be edited. (Users can follow a link by holding down the ALT key while clicking on the link.

    • datefield [#datefield(fieldname,fieldformat)]

      This macro is used to add date fields to the Template. Fields added using this macro are displayed with the icons used For Active Assembly. The fieldname parameter specifies the field to be added to the Template; the format parameter specifies the date format to use, which must conform to java.text.SimpleDateFormat.

    • displaydatefield [#displaydatefield (fieldname,fieldformat)]

      This macro is used to add date fields to the Template if the fields should not be available to be edited. Fields added using this macro are displayed without the icons used for Active Assembly. The fieldname parameter specifies the field to be added to the Template; the format parameter specifies the date format to use, which must conform to java.text.SimpleDateFormat.

    • field_if_set [#field_if_set(before,field,after)]

      This macro is used to add optional fields to the Template. It should be used for fields that are not required to have a value in the Content Editor. The fieldname parameter specifies the field to be added to the Template. The field data will be wrapped in the text specified by the beforetext and aftertext parameters.

    • datefield_if_set [#fdatefield_if_set(before,field,format,after)]

      This macro is used to add optional date fields to the Template. If should be used for date fields that are not required to have a value in the Content. The fieldname parameter specifies the field to be added to the Template. The format parameter specifies the format to used to render the field value. The field data will be wrapped in the text specified by the beforetext and aftertext parameters.

  • Rx Slot Macros

    This section lists macros used to add Slots to a Velocity Template. The following Slot macros are available:

    • simple slot [#slot_simple(slotname)]

      Displays the contents of the Slot, with no additional formatting or parameters. The slotname parameter specifies the Slot to add to the Template.

    • slot wrapped [#slot_wrapped(slotname,startslottext,endslottext )]

      Displays the contents of the Slot. Each Content Item in the Slot is wrapped in the text specified by the beforetext and aftertext parameters. The slotname parameter specifies the Slot to add to the Template.

    • full slot [#slot(slotname,header,before,after,footer,params)]

      Displays the contents of the Slot. The Slot as a whole is wrapped in the text specified by the header and footer parameters. Each Content Item in the Slot is wrapped in the text specified by the beforetext and aftertext parameters. The slotname parameter specifies the Slot to add to the Template. The params parameter specifies any parameters to use when processing the Slot.

    • raw slot [#initslot(slotname,params )]

      Inserts the raw Slot macro code. Used to create custom Slot processing in Templates.

    • node slot [#node_slot(node slotname, header, before, after, footer, params)]

      Used when both Velocity Templates and XSL Variants co-exist on the same system, generally to implement Managed Navigation. Works like the full slot macro, but uses the Content Item specified in the node parameter to expand the list of related Content Items in the Slot. Contents of #node_slot are not eligible to be modified using Active Assembly.

  • Rx Slot Miscellaneous Macros and Prebuilds

    This section lists miscellaneous macros and some prebuilt example templates. Miscellaneous macros include:

    • inner [#inner()]

      Used with Global Templates to add the page content to the Global Template output.

    • children [#children(childname,templatename,header,before,after,footer)]

      Used on Page Templates to add child editor data to the Page Template output. The childname parameter specifies the child to add to the Template. The templatename parameter specifies the Template to use to format the child editor output. The child table will be wrapped in the text specified by the beforetext and aftertext parameters. The table as a whole will be wrapped in the text specified by the header and footer parameters.

    Prebuilt example Templates include:

    • sample page
    • L-clamp Global Template
    • breadcrumbs nav
    • top nav
    • left nav

When you double-click on a macro, the Rhythmyx Workbench displays the Insert Template: macro dialog:

The Variables table lists the variables for the selected macro. Enter the value for the variable in the associated row of the Value column. The Rhythmyx Workbench adds this value to the Source field automatically. The Description of Variable field displays a description of the selected parameter.
