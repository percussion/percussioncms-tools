Maintaining Controls

A control is the mechanism or interface in a Content Editor through which users enter data into a field. Once the data is entered, the control displays it in the Content Editor. A control such as a rich text editor or a drop list may specify aspects of the data's format.

Rhythmyx supplies a set of default controls that are defined in the file <Rhythmyx root>/sys_resources/stylesheets/sys_Templates.xsl. In general, the default controls appear as options to all Content Editor fields. You can view the sys_Templates.xsl file, but should avoid editing it since it is overwritten on upgrade.

Upon installation, the file <Rhythmyx root>/rx_resources/stylesheets/rx_Templates.xsl is a duplicate of the sys_Templates.xsl file. Add or modify controls in rx_Templates.xsl, which is not overwritten on upgrade. Additions, modifications and deletions of controls in rx_Templates.xsl always take precedence over the definitions of controls in sys_Templates.xsl.

To view the default stylesheet for Rhythmyx controls:

  1. In the UI Elements Design view, under the Content Editor folder, right click System Controls and choose Open With > [editor] in the drop menu.

    sys_Templates.xsl opens in the editor you have chosen.

To view or modify the overriding stylesheet for Rhythmyx controls:

  1. In the UI Elements Design view, under the Content Editor folder, right click User Controls and choose Open With > [editor] in the drop menu.

    rx_Templates.xsl opens in the editor you have chosen.

  2. Make any modifications or additions to the file. For information about editing the rx_Templates.xsl file, see the Technical Reference Manual.
