Field Visibility Editor

The Field Visibility editor allows you to add a visibility rule to a field and maintain the visibility rule.

To access the Field Visibility editor:

  • When creating or editing a field in the Field Properties editor, click the [Visibility] button.

    Screenshot of Field Visibility Editor

Field Descriptions

Visibility - Table of visibility rules.

Rule - Visibility rule set up in the Rule Details section. All visibility rules are user-defined functions or manually entered conditions. Visibility rules are listed in the order in which they are selected.

Boolean - Relationship to next rule in table. Value is AND or OR.

Rule Details - Section listing the Rule type.

Rule type -

Rule type - Choices are:

  • Create Only - Specifies that the field is read only under the conditions specified during creation only.
  • Modify Only - Specifies that the field is read only under the conditions specified during modification only.
  • JEXL - When selected, the editor opens a text box for entering a JEXL expression that defines the conditions under which the field is read only.
  • Conditional - When selected, the editor opens a table for defining conditions.
  • Extension - When selected, the editor opens a Name drop list. The drop list includes extensions that define when the field is visible.

Conditional table -

Variable - The variable being tested. Use the Value Selector to specify the variable.

Op - The boolean operator for the condition.

Value - The value against which you are testing the Variable. Use the Value Selector to specify the value.

Boolean (in Rule) -Boolean operator joining and evaluating multiple conditions. Options are AND or OR, or null.

Name - Name of the UDF that applies a visibility rule to the field. If the UDF includes extensions, the Extensions Parameters dialog opens.
