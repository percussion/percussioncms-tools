Field Properties Editor

The Field Properties editor allows you to configure field properties that are set or modified less frequently than those on the main Field and Field Set editor.

To access the Field Properties editor:

  • In the Field and Field Set editor, select a field in the table and click [All Properties].

Using the Field Properties Editor you can:

  • Configure additional properties
  • Edit properties

    Screenshot of Field Properties Editor

Field Definitions

Field name - Read-only. The system name of the field.

Data type - Read-only. The type of data that the field stores, such as text or binary; see Default Content Editor Controls for the default data type associated with each control.

Storage size - The storage size of the field in the backend table; see Default Content Editor Controls for the default storage size associated with each control.

Default value - The default value of the field, if any (entered from the previous screen).

Mime type mode - Indicates how the mime type to be associated with the field is chosen. The full-text search engine uses the mime type to correctly extract text from the field. The possible values for Mime type mode are:

Default - When you choose Default, the system sets the field's mime type according to the Data Type and Storage size. When the Data Type is binary, Rhythmyx maps the uploaded file's extension to a Mime type value. (Mappings are stored in the file <RxRoot>/rxconfig/Server/mimemap.properties. You can add your own mappings to this file)

From Selection - When you choose From Selection, the Mime type value field becomes a drop list of all mime types included with the full-text search engine and any added to your system through extensions. This option lets you choose any of the mime types in the drop list to associate with the field.

From Extension Field - Only choose From Extension Field for binary Content types. When you choose this option, the Mime type value field becomes a drop list of text fields in the Content Type. This option lets you choose the field that holds the uploaded file's extension so Rhythmyx can map it to a mime type.

From Mime Type Field - When you choose From Mime Type Field, the Mime type value field becomes a drop list of text fields in the Content Type. This option lets you choose the field that holds the mime type.

Mime type value - If you choose Default in the Mime type mode field, Mime type value is grayed out. If you choose any of the other choices, Mime type value becomes a drop list of values. See the descriptions of each Mime type mode choice in the preceding definition for a description of the options in the drop list associated with each choice.

For more information about how the search engine uses mime types in determining how to extract text, see the Rhythmyx Server Administrator online help.

Label - The display name for the field.

Mnemonic - The key used in the ALT + key sequence for moving the cursor to this field; drop list includes all unused letters for fields in the Content Type.

Error label - Sentence or phrase to display if field entry is not valid.

Control - The mechanism for displaying the field, such as drop-list or text box. See Default Content Editor Controls for the default controls.

Show in summary - Enabled for fields in child field sets only. If you want a field name and its values to display in the main Content Editor when users open it, leave Show in summary checked. If you uncheck Show in summary, users must click [Edit Table] in the Content Editor to view the field (and all other fields in the field set) in a table.

Show in preview - Specifies whether or not to display this field in a preview of a Content Item.

Show "clear field" text box in binary control - Enabled if data type for field is binary. When checked, the editor displays a box the user can check to clear the database table column before uploading binary data. If the "clear field" text box is not checked, the contents of the column in the backend table are not changed if the field is left blank in the Content Editor.

Search:

Allow this field to be searched - Indexes the contents of the field for searching.

Include in full-text multi-field query - When checked, field is automatically searched during a user-entered search. When unchecked, field is not searched during a user-entered search unless it is specified in the Advanced Search dialog. For example, you might not want the Workflow to be included in a Search, so you might uncheck the box for the Workflow field. Checked by default.

[Read Only] - Opens the Field Editability editor. This editor lets you configure whether or not a field is read only.

[Visibility] - Opens the Field Visibility Rule editor. This editor lets you configure whether to show or hide the field in the Content Editor.

[Validation] - Opens the Field Validation editor. This editor lets you configure validity checks (validation rules) on the format and value of data entered into the field.

[Transforms] - Opens the Field Transforms editor. This editor lets you specify how the Content Editor changes the format of data entered into the field. Note: In versions of Rhythmyx prior to 6.0, field transforms were referred to as field translations.
