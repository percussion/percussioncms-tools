Setting Rhythmyx Workbench Preferences

The Preferences dialog is an Eclipse dialog for setting user preferences. In Rhythmyx the Preferences dialog includes a Rhythmyx node with a Security sub-node.

To set Rhythmyx Workbench Preferences:

  1. In the Menu bar, choose Window > Preferences.

    The Preferences dialog opens.

  2. Click the Rhythmyx node to display general Rhythmyx preferences.

    Screenshot of Preferences dialog

Field Definitions

Show warning for read-only objects on open - When users attempt to open objects to which they do not have write-access, a dialog asks them if they want to open the object as read-only. To automatically open the object in read-only mode without displaying the dialog, uncheck this option. Checked by default.

Show deprecated functionality - When checked, deprecated java extensions are shown. Unchecked by default.

Automatically connect when Rhythmyx Workbench opens - If a connection in the Connection dialog is marked as default, logs user in to the Workbench without opening the Connections dialog. Checked by default.

Automatically activate 'Field Problems' view upon field error - If an error is present in a field in a Rhythmyx editor, the Field Problems view automatically opens and displays a description of the error. Checked by default.

Click on the Security subnode to display the Security preferences page.

Show legacy interfaces for content type extensions - When a user adds a read-only, validation, visibility or transform rule to a field in a Content Editor field, there is an option to choose an extension to add as a rule. When the Show legacy interfaces for content type extensions box is unchecked, only udfs specific to the type of rule being created are available in the extension drop list. When the Show legacy interfaces for content type extensions box is checked, all udfs are available in the extension drop list, as they were in the Workbench prior to Rhythmyx 6.0. By default, this box in unchecked in new installations, and checked in installations that upgrade from a version prior to 6.0.
