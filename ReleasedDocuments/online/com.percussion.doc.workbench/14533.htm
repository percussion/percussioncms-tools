Default Object ACL Settings

In the Preferences dialog, click on the Security subnode to display the Security preferences page:

Screenshot of Default Object ACL Settings Dialog

To set access using this dialog, see Editing Object Access.

Field Definitions

Default Object ACL - A list of all Roles, Communities, and Users who have a default access level configured for all objects in the Rhythmyx Workbench. There are two special entries that appear by default:

Default - Defines permissions for a user not represented by another entry in the list. By default all Design access permissions are checked.

AnyCommunity - Defines permissions for a Community that is not listed (for objects that are associated with Communities). By default, Runtime access (Read) is checked.

[Add ACL Entry] - Opens the Add ACL Entry dialog, which lets you add a new Role, Community or User to the default object ACL.

[Delete] - Deletes the selected entry

Design Access - Permissions for working on all objects in the Rhythmyx Workbench. Users are given the maximum access of all checked permissions.

Read - Permission to view the settings for all objects in the Rhythmyx Workbench.

Update - Permission to change the settings for all objects in the Rhythmyx Workbench.

Delete - Permission to delete all objects in the Rhythmyx Workbench.

Modify ACL - - Permission to modify the permissions for all objects either in this dialog or in the Community Visibility view.

Runtime Access - This field only appears for objects that can be associated with Communities. These objects are Content Types, Display Formats, Menus, Menu Entries, Searches, Sites, Templates, XSL Variants, Views, and Workflows. Runtime access gives access to this object in Content Explorer.

Read - Permission to have access to this object in Content Explorer.

[Restore Defaults] - Restores the default access settings for each entry in the default object ACL.

[Apply] - Save changes without closing the dialog.
