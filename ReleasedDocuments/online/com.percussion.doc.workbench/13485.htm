Display Control Properties Editor, Control Tab

The Control tab of the Display Control Properties editor displays the parameters for the control and the settings for its inline link flags and MSM identifier flag.

Use this tab to:

Add parameters and set the inline link and MSM flags.

Edit parameters and the settings of the inline link and MSM flags.

Note: If the Display Control Properties dialog only includes this tab, the tab is not shown.

Screenshot of Display Control Properties Editor, Control Tab

Field Descriptions

Control Name - Default control or user-defined control chosen from drop list.

Data Type - Read only. The data type entered in the Field Properties editor.

Parameters - If required, parameters for the control.

Name - Drop list of all required parameter names for the control.

Value - Value of parameter. Either enter a literal value or click [...] to enter a User Defined Function using the Function Properties dialog or an Other Value using the Value Selector.

Field data properties

Allow inline links - Allows inline links in a field that uses the sys_EditLive, sys_EditLiveDynamic, or sys_EditBox control.

Clean up broken inline links - Removes inline references to purged Content Items for fields that use the sys_EditLive, sys_EditLiveDynamic, or sys_EditBox control.

May contain identifiers - Used by Multi-Server Manager. The value of the field may include identifiers of other objects in your CMS. For example, if the control is used for the URL of the query for an auto index, the parameters in the URL may include a Template ID. If you flag this field by checking May contain identifiers you let Multi-Server Manager discover these identifiers when building deployment archives.

Namespaces

Cleanup namespaces Specifies whether namespaces should be cleaned up selectively when the field is uploaded to the Repository and when the field data is included in assembled output. Specify namespaces that should not be cleaned up in the Configured namespace prefixes field.

Allow active tags Specifies that ASP and JSP tags that are included in the field data should be retained in the data and not cleaned up.

Configured namespace prefixes List of namespaces that should note be cleaned up during text cleanup before field data is updated to the Repository and during assembly. Use commas to separate namespaces.
