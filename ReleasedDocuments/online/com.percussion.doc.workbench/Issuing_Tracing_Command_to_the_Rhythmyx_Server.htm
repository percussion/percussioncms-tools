Issuing Tracing Commands to the Rhythmyx Server

You can manage tracing by issuing tracing commands directly to the server using the admininstration page. To access the administration page, open a browser and in the Address field, enter:

http://<host:port/Rhythmyx/admin/console.jsp

Where

host is the name or IP address of the machine where the Rhythmyx server resides; and

port is the Rhythmyx port (9992 by default).

You must log in to access the administration page.

To manage tracing, enter the following command:

Trace [help] [[type_flags|'default'] appName]

Where:

[help]- if specified, other parameters are ignored.  Equivalent to specifying no parameters.  Produces a list of all flags and the tracing they enable.

[type_flags]- A sequence of 1 to 4 OR'ed sets of flags that indicate the tracing options you want to turn on.  Specify flags as hexadecimals (with a leading 0x).  See Tracer Types and Flags for details about the trace options associated with each flag.  If tracing is already enabled, it is re-initialized with the new options.  

default - enter "default" for a flag set if you want to use the trace settings already set in the application for that flag set.  Generally, you would use "default" instead of type flags if you changed the trace settings and wanted to return to the original settings.

AppName - the name of the application you want to trace.  If you specify one or more type_flags, Rhythmyx turns on tracing for the application.  If you do not specify any flags, Rhythmyx displays the current state of the tracing flags; if Rhythmyx returns only zeroes, tracing is not currently enabled for this application.  If the specified application does not exist or is not running, Rhythmyx returns an error message informing you about it.  In that case, tracing is not enabled for the specified application.
