Maintaining Controls

A control is the mechanism or interface in a Content Editor through which users enter data into a field. Once the data is entered, the control displays it in the Content Editor. A control such as a rich text editor or a drop list may specify aspects of the data's format.

Rhythmyx supplies a set of default controls that are defined in the file <Rhythmyx root>/sys_resources/stylesheets/sys_Templates.xsl. In general, the default controls appear as options to all Content Editor fields. You can view the sys_Templates.xsl file, but should avoid editing it since it is overwritten on upgrade.

To view the default stylesheet for Rhythmyx controls, in the UI Elements Design view, under the Content Editor folder, right click System Controls and choose Open With > [editor] in the drop menu. The stylesheet sys_Templates.xsl opens in the editor you have chosen.

Custom controls can be implemented in two ways.

Custom controls that are to be included in a package must be defined in XSL files in the directory <CMServerroot>/rx_resources/stylesheets/controls. Each custom control must have a unique XSL file, and the XSL file should have the same name as the control; for example, if you wanted to make a new calendar control named rff_newCalendar, the XSL file containing the control would be rff_newCalendar.xsl. All controls provided by Percussion Software begin with the string "sys_"; for example, sys_DatePicker.  User-developed controls should not begin with this prefix.

To view or modify a custom control that can be included in a package, choose the System Design tab, expand CMS Files > rxresources > stylesheets > controls, right-click on the control on which you want to operate and choose Open With > [editor] in the drop menu. The stylesheet for the control you selected opens in the editor you specified.

Legacy custom controls can be implemented in the file <CMServerroot>/rx_resources/stylesheets/rx_Templates.xsl. Controls implemented in this file cannot be included in a package. To view or modify controls defined in this file, in the UI Elements Design view, under the Content Editor folder, right click User Controls and choose Open With > [editor] in the drop menu. The file rx_templates.xsl is returned in the editor you specify.

For details about creating and maintaining controls, see the Technical Reference Manual.
