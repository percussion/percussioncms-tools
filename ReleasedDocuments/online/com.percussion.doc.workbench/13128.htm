Maintaining Templates

A Template is a Rhythmyx object that defines the assembly processing to generate an output. The key element of the Template definition is the Assembly plugin, which defines the assembly mechanism used for the Template. The following Template plugins are available with Rhythmyx:

  • Velocity

    Produces a formatted text output, such as an XML snippet or HTML page; requires a Velocity template to define the output formatting.

  • binary

    Assembles binary objects for output. This assembler requires the bindings $sys.binary and $sys.mimetype.

  • database

    Assembles output for delivery into database tables

  • Content Assembly application

    Uses a Rhythmyx Content Assembly XML application to extract Repository data and produce the formatted output.

A Template may also include an ordered set Bindings. Each Binding consists of a named variable whose value is a JEXL expression defining the processing for that binding. The Bindings are executed in the order in which they are defined.

For details about JEXL expressions, see http://jakarta.apache.org/commons/jexl/

For detailed information about implementing Templates, see "Creating Templates" in the Rhythmyx Implementation Guide.

Use the New Template wizard to create a new Template.

Use the Template editor to complete a Template or to maintain an existing Template.

You can:

  • create a new Template;
  • modify an existing Template; or
  • delete a Template.
