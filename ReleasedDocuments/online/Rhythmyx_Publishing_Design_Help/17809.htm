Defining a New Site

To define a new Site:

  1. In the Navigation pane, select the Sites node, and from the Menu bar, choose Action > Create Site.

    Rhythmyx displays the Site editor in the View and Edit pane.

  2. Enter a Name for the Site. The name must begin with a letter, and can contain any alphanumeric characters, underscores, hyphens, or dots (periods). Optionally, enter a free-form text Description of the Site.
  3. Enter the Rhythmyx Path.
    1. Click the Browse button next to the Source Folder Root Path

      Rhythmyx displays the Site Folder Browser.

    2. Browse to the Folder you want to use and the root Folder for the Site. Note that the Site Root Folder must be a child of the //Sites node, but need not be an immediate child of that node. In other words, both //Sites/EnterpriseInvestments and //Sites/Investments/EnterpriseInvestments are valid Site root paths).
    3. In the menu bar, click Done.

      Note that you can also enter the Site Root Folder Path manually. The specified path must begin with //Sites, and must exist under that node in the Navigation pane of Content Explorer. If the path does not exist, you cannot save the Site.

  4. Select the Global Template to use when formatting Page Templates for the Site. Options include all Global Templates defined in the system.
  5. Enter the Published URL. This value is the URL used to access the published Site.
  6. Optionally, enter the Published Path.

    If you are defining a Site to publish to a database, enter the connection information (need to get the correct connection information for each RDBMS.

  7. To enter special Site details, click the Show Site Details link.

    The Site editor displays the Site details fields.

    1. If you are defining a Site to use an FTP or SFTP Delivery Type, enter the FTP Server IP Address, FTP Port, User Name, and Password.
    2. To define a new unpublish flag, enter the character (case-insensitive) in the Unpublish Flags field. Multiple characters can be added, separated by commas. (Note: for each character you enter in this field, you should also define a Keyword Choice to the Publishable Keyword whose value matches the character you added to the field. For details see "Configuring Unpublish Flags" in the Rhythmyx Technical Reference Manual. You should also define a Workflow State to use each character.)
    3. The Nav Theme field is used in systems that use Managed Navigation as implemented in Rhythmyx Version 5.5 through 5.7. Specify the Nav Theme you want to use for the Site in this field.
    4. If there are XML namespaces you do not want removed when Content Items are uploaded, enter them as a comma-delimited list in the Allowed Namespaces field. Note that entries in this field should have a corresponding namespace definition in the file <Rhythmyxroot>/rxconfig/server/stylesheetCleanupFilter.xml. If a namespace specified for the Site does not have a corresponding entry in the that file, it will be filtered out
  8. To add a Context Variable:
    1. In the Menu bar, choose Action > Add Context Variable.

      Rhythmyx displays the Add Context Variable page.

    2. Enter a Context Variable Name.
    3. Enter a Context Variable Value.
    4. Choose the Context in which to use the Context Variable.
    5. To copy an existing Context Variable, click the copy button .
  9. To add a new Context Variable:
    1. In the Menu bar, choose Action > Add Context Variable.

      Content Explorer displays the New Context Variable editor.

    2. Enter the Context Variable Name.
    3. Enter the Context Variable Value.
    4. Choose the Context to which the Value will apply.
    5. Top copy an existing Context Variable, click the copy button in the row of the Context Variable you want to copy. Rhythmyx copies the data for the Context Variable into the appropriate fields. Change the Context Variable Value and Context as needed. Note that the same Context Variable Name may be used several times with different values for different combinations of Site and Context, so you may want to to copy the instance that most closely matches the values you need.
    6. In the Menu bar, click Done.

      Content Explorer returns to the Site editor.

  10. To save the Site, in the Menu bar, Click Save.
