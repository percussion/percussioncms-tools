Maintaining Delivery Type Registrations

Publishing processing is performed by Delivery Handlers. Rhythmyx ships with four standard Delivery Handlers (File System, FTP, SFTP, and Database), but you can implement custom Delivery Handlers if the standard handlers do not meet your needs. For details about implementing a custom Delivery Handler (including registering the Spring bean), see "Implementing a Delivery Handler" in the Rhythmyx Technical Reference.

After implementing a custom Delivery Handler, you must register it as a Delivery Type to make it available for assignment to a Site. (Note, you can register multiple Delivery Types using the same Spring bean.)

Use the Delivery Type editor to create and maintain Delivery Type registrations. You can:

  • Create a Delivery Type registration;
  • Modify a Delivery Type registration;
  • Delete a Delivery Type registration.
