Modifying an Edition

To modify an Edition:

  1. In the Navigation pane, expand the Sites node, expand the Site with which the Edition you want to modify is associated, and click on the Edition.

    The View and Edit pane displays the Edition editor.

  2. You can change the Name or Description of the Edition.
  3. You can change the Priority of the Edition. Options are:
    • Highest
    • High
    • Medium
    • Low
    • Lowest

    An Edition specified with a higher priority interrupts the processing of any currently running Edition that is specified with a lower priority. Processing of an Edition with a lower priority does not begin until processing of Editions with higher priority is complete.

  4. You can change the Behavior of the Edition. Options are:
    • Publish

      Clears the Site record, then publishes the Content Items generated by the Edition Content Lists. No unpublishing processing. Choose this option if you want to implement legacy publishing behavior, which required an explicitly defined unpublish Content List.

    • Unpublish then Publish

      Unpublishes Content Items that have been archived or removed, then publishes any Content Items generated by the Edition Content Lists.

  5. To add Content Lists to the Edition:
    1. In the Menu bar, choose Action > Add Content List Association.

      Rhythmyx displays the Add/Edit Content List page in the View/Edit pane.

    2. Enter filter criteria to filter the list of Content Lists. Check the Limit to Site and Unassigned checkbox to limit the list to Content Lists already associated with the Site of the Edition or not associated with any Site. Enter a filter string into the Filter Content Lists field to filter the list to Content Lists that match the string. The filter string should be the first few characters of the Content List name. When you click the [Apply] button, the filter will be applied and only Content Lists that match the criteria you specified will be listed.
    3. Check the boxes of the Content Lists you want to add to the Edition.
    4. Select the Assembly Context to use when creating links between Content Items when publishing this Edition..
    5. Select the Publishing Context to define the output location for pages generated by the Edition.
    6. If you your system uses AuthTypes (used in legacy systems originally implemented in Rhythmyx Version 5.7 or earlier), specify the AuthType for the Edition. (If your implementation is based on Rhythmyx Version 6.0 or later, you can ignore the AuthType field.)
    7. Click the [OK] button to add the Content Lists to the Edition.

      The Content Lists you selected will be added with the properties you specified.

    8. Repeat Steps a through g to add more Content Lists to the Edition.
  6. To modify the data of a Content List's association with an Edition:
    1. Select the radio button in the row of the Content List association you want to modify, and in the Menu bar choose Action > Edit Selected Content List Association.

      The View and Edit pane displays the Add/Edit Content List dialog.

    2. You can change the Assembly Context, Delivery Context, or AuthType. You can also add more Content Lists using the same properties.
    3. Click the [OK] button to save your changes.
  7. To add a new task:
    1. In the Menu bar, choose Action > Add Pre Task to add a Task to run before Edition processing begins or Action > Post Task to add a Task to run after Edition processing is complete.

      Rhythmyx adds a blank task to the Edition in the appropriate table. (The table is added if no Tasks have been defined previously.

    2. If you want processing of the Edition to continue if the task fails, check the Continue on Failure checkbox. Otherwise, leave this box unchecked.
    3. Select the task Extension you want to execute. If the task extension has parameters, the Edition editor displays the parameters below the extension field. Enter values for the parameters.

      Currently, the only task extension available is sys_editionCommandTask, which has one parameter, a command entered in the Command field displayed when you select this extension. Substitution variables in the command take the form $[variable]. The following variables are available:

      • edition_name (Name of the Edition)
      • site_name (Name of the Site associated with the Edition)
      • site_publishing_path (Path to which the Edition output was published. Derived from the Published Site Root Path on the Site editor.)
      • edition_start_time (Date/timestamp of the initiation of Edition processing)
      • edition_end_time (Date/timestamp of the completion of Edition processing. Only valid for post-edition tasks.)
      • site_homepage_url (The URL used to navigate to the root of the published Site. Derived from the Published Site Home URL field on) the Site editor.
      • success_count (Count of the number of Content Items published successfully. Only valid for post-edition tasks.)
      • failure_count (Count of the number of Content Items for which publishing failed. Only valid for post-edition tasks.)
  8. To modify a Task, change the value in any field of the task definition..
  9. To change the order of Content Lists or Tasks, select the radio button in the row of the Content List or Task you want to move. To move the selected row up, in the Menu bar choose Action > Move Selected Row Up. To move the selected row down, in the Menu bar choose Action > Move Selected Row Down.
  10. To delete a Content List Association or Task, select the radio button in the row of the Content List or Task you want to delete, and in the Menu bar, choose Action > Delete Selected Row.
  11. To save your edits, in the Menu bar, choose [Save].
