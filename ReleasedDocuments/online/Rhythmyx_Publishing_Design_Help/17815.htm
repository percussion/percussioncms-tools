Editions Editor

Use the Editions editor to create and maintain Editions.

To access the Editions editor:

  • in the Navigation pane, select the Sites node (or any of its child nodes) and in the Menu bar, choose Action > Create Edition;
  • in the Navigation pane, expand the Sites node, expand the Site associated with the Edition you want to edit, select the Edition, and in the Menu bar, choose Action > Edit Selected Edition;
  • in the Navigation pane, expand the Sites node, expand the Site associated with the Edition you want to edit, and click on the Edition you want to edit.
  • in the Editions List, double-click on the Edition you want to edit.

Field Descriptions

Name Name of the Edition. Edition names must be begin with a letter, and can contain any alphanumeric characters, underscores, hyphens, or dots (periods).

Description Free-form text description of the Edition. Value can contain any alphanumeric characters and punctuation marks.

Priority Processing priority of the Edition. Options are:

  • Highest
  • High
  • Medium
  • Low
  • Lowest

Processing of an Edition with a higher priority interrupts processing of Editions with lower priority. For example, if an Edition assigned a priority of Medium is currently running and an Edition with a priority of High is started, processing of the Medium priority Edition stops until processing on the High priority Edition is complete. If another Edition with Low priority is also started, the system does not start processing it until processing of the High and Medium priority Editions is complete.

Typically, full Editions are assigned lower priority (Low and Lowest), incremental Editions are assigned Medium priority, and demand Editions (Publish Now) are assigned higher priority (High and Highest).

Behavior Specifies the publishing behavior of the Edition. Options include:

  • Publish

    Clears the Site record, then publishes the Content Items generated by the Edition Content Lists. No unpublishing processing. Choose this option if you want to implement legacy publishing behavior, which required an explicitly defined unpublish Content List.

  • Unpublish then Publish

    Unpublishes Content Items that have been archived or removed, then publishes any Content Items generated by the Edition Content Lists.

Content List Table Table listing the Content Lists associated with the Edition. You can re-order the list of Content Lists list or removed Content Lists from the list. Double-click on a Content List in the table to display a preview of the Content List results.

Pre Tasks and Post Tasks

Pre Tasks are processed before Edition processing is initiated. Pre Tasks are processed in the order in which they are defined in the Pre Tasks table.

Post Tasks are processed after Edition processing is complete. Post Tasks are processed in the order in which they are defined in the Post Tasks table.

Continue on Failure Checkbox Check to box to indicate that Edition processing should continue if the task fails. If the box is not checked and the task fails, Edition processing stops.

Extension The task extension to run.

Currently, the only option is sys_editionCommandTask, which has one parameter, a command entered in the Command field displayed when you select this extension. Substitution variables in the command take the form $[variable]. The following variables are available:

  • edition_name

    Name of the Edition

  • site_name

    Name of the Site associated with the Edition

  • site_publishing_path

    Path to which the Edition output was published. Derived from the Published Site Root Path on the Site editor.

  • edition_start_time

    Date/timestamp of the initiation of Edition processing

  • edition_end_time

    Date/timestamp of the completion of Edition processing. Only valid for post-edition tasks.

  • site_homepage_url

    The URL used to navigate to the root of the published Site. Derived from the Published Site Home URL field on the Site editor.

  • success_count

    Count of the number of Content Items published successfully. Only valid for post-edition tasks.

  • failure_count

    Count of the number of Content Items for which publishing failed. Only valid for post-edition tasks.
