Maintaining Sites

A Site defines a location where output will be published. In most cases, a Site is either a directory location (which may be accessed directly or through FTP) or a database. You can maintain multiple Sites on a single Rhythmyx server. Different Sites usually refer to different output locations.

You can define a number Site properties. These properties are available to all Sites, but each property takes a unique value for a specific Site.

When you select the Sites node in the Navigation pane of the Publishing Design tab of Content Explorer, Rhythmyx displays the Site List in the View/Edit pane. The Site List lists all of the Sites currently defined in the system. Clicking the header of any column in the table sorts the rows in the table based on the order of that column.

Use the Site editor to create and maintain Sites. You can:

  • create a new Site;
  • copy a Site;
  • modify an existing Site;
  • delete a Site.
