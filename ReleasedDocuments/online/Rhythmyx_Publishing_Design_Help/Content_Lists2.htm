Content Lists

A Content List is a Rhythmyx object that defines the rules that determine which Content Items to extract from the database for Publishing.  Divide a Site into separate Content Lists to give site managers control over the way the Publication looks on the Site.

Generally, the following Content Lists are defined for each Site:

  • A Full Content List of all non-binary Content Items

    Publishes all Content Items that do not manage binary files; generally run on a regular basis (such as weekly)

  • A full Content List of all binary Content Items

    Publishes all Content Items that manage binary files; generally run less frequently than non-binary full publish Content Lists because binary Content Items take longer to publish than non-binary data (in other words, text).

  • An Incremental Content List of all Content Items

    Publishes only Content Items that were modified since the last publishing run. Generally run frequently (daily or sometimes hourly)

    Note that in systems upgraded from Rhythmyx version 6.5 or earlier, additional Content Lists were required to unpublish Content. Separate unpublish Content Lists are no longer necessary in Rhythmyx, as unpublishing is handled by the Edition.

A Content List consists of the following data:

  • The Delivery Type of the Content List

    The Delivery mechanism that will be used to delivery the assembled output of the Content List.

  • The Type

    Specifies whether the Content List is Normal or Incremental. An Incremental Content List publishes only Content Items that were modified since the last publishing run (for example, the Content Item went Public, or some of the Content Item data was modified).

  • The Content List Generator

    A Rhythmyx extension used to define the list of Content Items to publish. Three Content List Generators are available:

    • sys_PublishedSiteItems

      This Content List Generator generates a list of all Content Items published on a Site. Must be used in conjunction with the sys_SiteTemplateExpander. Usually used for unpublishing.

    • sys_SearchGenerator.

      This Content List Generator generates the list of Content Items to publish based on a JSR-170 query, which is defined in the query parameter. This Content List Generator should be selected in most cases.

    • sys_SelectedItemsGenerator

      This Content List Generator generates a list of Content Items to publish based on a set of selected Content Items. It is used in on-demand publishing, and usually should not be selected.

  • A Template Expander

    A Rhythmyx extension that generates a list of Templates to use to format the Content Items in the Content List. Two Template Expanders are available:

    • sys_SiteTemplateExpander

      Publishes all Templates used on the Site being Published. Requires the siteid parameter, which specifies the Site whose Templates to use.

    • sys_ListTemplateExpander

      Publishes only the listed Templates. Requires the template parameter, which specifies the Templates to use when publishing. Generally used for Database Publishing. (NOTE: The listed Templates will be published regardless of the Publish option specified on the Template or of the visibility of the Template to the Site being published.)

  • An Item Filter

    Specifies the Item Filter to use to filter the related Content Items to be published. An Item Filter is a set of rules used to filter the Content Items to be published. Item Filters are defined in the Rhythmyx Workbench.

Use the Content List editor to create and maintain Content Lists. You can:

  • create a new Content List;
  • modify an existing Content List; or
  • delete a Content List.

Note that a separate dialog is available for maintaining legacy Content Lists (Content Lists implemented in Rhythmyx Version 5.7 or earlier that use an XML application to assemble the Content).

Two List Content Lists dialogs are available. When you click on the Content Lists node under a Site, the List Content List dialog lists all Content Lists associated with an Edition for the Site. When you click on the Content Lists node at the root level of the Navigation pane, the List Content Lists dialog lists only Content Lists not associated with any Edition.
