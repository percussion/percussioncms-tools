Content List Editor

Use Content List editor to create and maintain Content Lists.

To access the Content List editor:

  • While viewing the List Content Lists dialog, in the Menu bar, choose Action > Create Content List.
  • in the Navigation pane, expand the Sites node, expand the Site with which the Content List you want to modify is associated, expand the Editions node, and double-click on the Edition with which the Content List you want to modify is associated.

Field Descriptions

Name Name of the Content List. Content List names must be begin with a letter, and can contain any alphanumeric characters, underscores, hyphens, or dots (periods).

Description Free-form text description of the Content List. Value can contain any alphanumeric characters and punctuation marks.

Content List Type Specifies whether the Content List is Normal or Full.

  • A Normal Content List clears the Site record, removes all existing Content Items from the target location, and publishes all Public Content Items in the Site.
  • An Incremental Content List removes any expired Content Items from the Site, updates any Content Items that have changed since the last publishing run, and publishes any Content Items that became Public since the last Publishing run.

Delivery Type The Delivery Type to use to deliver published Content Items to the output location. Options include all Delivery Types defined in the system (by default, File System, FTP, SFTP, Database).

Item Filter The Item Filter to use to filter the original list of Content Items to produce a final list of Content Items to be published.

Generator The Generator to use to select that base list of Content Items to be published. This list is processed by the Item Filter to produce the final list. Options include all Generators registered in the system; the default options include:

  • sys_Search

    Generates a list based on a JCR query.

  • sys_PublishedItems

    Outputs the list of Content Items currently published on the Site; must be used with the sys_SiteTemplateExpander. Used primarily for unpublishing in legacy systems implemented on Rhythmyx Version 6.5 or earlier.

  • sys_SelectedItems

    Returns the list of selected Content Items. Used in demand publishing.

In most cases, when a Generator is selected, one or more parameter fields will be displayed in the Content List editor where you can enter Generator parameters. The specific parameters depend on the Generator selected.

Template Expander Extension that selects the Templates to use to assemble the Content Items. Options include all Template Expanders registered in the system; the default options include:

  • sys_SiteTemplateExpander

    Publishes all Templates associated with the Site being published.

  • sys_ListTemplateExpander

    Publishes only the listed Templates.

In most cases, when a Template Expander is selected, one or more parameter fields will be displayed in the Content List editor where you can enter Template Expander parameters. The specific parameters depend on the Template Expander selected.

Base Resource Path (Advanced option) Path to the Content List servlet. Defaults to /Rhythmyx/contentlist. Should only be changed if you implement an alternate Content List servlet. For details about implementing servlets in Rhythmyx, see the Rhythmyx Technical Reference Manual.

Extra Parameters (Advanced option) Additional HTML parameters to include in the request to the Servlet. Parameters must be coded as HTML parameters.
