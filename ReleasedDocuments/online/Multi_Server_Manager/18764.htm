Glossary of Terms

Advanced Transforms

Option in Install Archive wizard that enables you to remap elements/datasources to to other elements/datasources (including those that are not in the archive) so that you can create new mappings with the freed elements/datasources.

Ancestors

Elements, such as files or exits, that are not part of a package but are affected by its dependencies. For example, if a Content Type is associated with a Community, the Community is an ancestor of that Content Type's package because it is not part of the package but is affected by changes to its elements.

Archive

The Multi-Server Manager file that contains elements to install. The archive is created from elements on the source server.

Archive Summary

Dialog that shows installation details of an archive and its packages. You can access the Archive Summary through the archives view in the details pane or through the last dialog that the Install Archive wizard shows after a successful installation.

Archives View

When you click an archives node in the Server Groups Tree, the Multi-Server Manager displays the archives view in the details pane. For each archive installed on the server, archives view displays: archive name, installation date, source server, Rhythmyx version, and Rhythmyx build.

Create Archive/Descriptor Wizard

A wizard that guides you through the process of defining a descriptor and/or creating an archive.

Custom Create Mode

Mode of the Create Archive/Descriptor wizard that takes you to the Identify ID Types dialog to review and modify your ID types whenever you are installing server objects with hard-coded identifiers.

Custom Installation Mode

Mode of the Install Archive wizard that always takes you to the Transforms (mapping) dialog to review or modify your mappings.

Dependencies

Elements that make up a package, including the main element and other elements that it requires to function properly. For example a Display Format package includes the Display Format definition and the Display Format's ACL definition.

Descriptor

File that stores information for creating an archive.

Descriptors View

When you select a Descriptors node in the Server Groups Tree, the Multi-Server Manager displays Descriptors view in the Details Pane. Descriptors view displays the name, description, and last modified date for each descriptor on that server.

Details Pane

Panel on Multi-Server Manager that displays a view with information about the node (descriptor, archive, or package) selected in the Server Groups Tree.

Guess All

Button on the Element Maps dialog that causes Multi-Server Manager to create all of the mappings for you.

ID Types

The types of elements represented by ID numbers that the Multi-Server Manager (MSM) finds in code associated with other elements.

The user must identify what ID types (if any) these numbers represent. This is necessary so MSM can package an element with code that includes an ID type. For example, if MSM knows that an application includes code that references the number 3, and that this represents Template 3, it can suggest that Template 3 be included in the package.

Install Archive Wizard

A wizard that guides you through the process of installing an archive from a source server onto a target server.

Local Dependencies

Dependencies that make up the package for an element and must be installed with it.

Manifest

File that describes the contents of an archive. The Multi-Server Manager creates a manifest with an archive and you are able to save the manifest to a file.

Mapping Datasources

The process of indicating to the Multi-Server Manager which non-Rhythmyx datasource on the target server should replace a non-Rhythmyx datasource on the source server.

Mapping Elements

The process of indicating to the Multi-Server Manager which element on the target server is equivalent to an element on the source server.

Mappings

Matches that the user makes between elements or datasources on the source and target servers so that the Multi-Server Manager knows which entities on the source server represent entities with different names or ID numbers on the target server.

Multi-Server Manager

Rhythmyx client that enables you to move Rhythmyx elements from one Rhythmyx server to another.

Package

A group of associated elements that are packaged together and included in an archive. Usually a package consists of a main element and its dependencies.

Package Log

After you install an archive, you can view the package log, which shows the dependencies installed, transforms, dependency validation warnings, and the installation actions on each element in an installed package. You can access the package log through the Archive Summary dialog or through the packages view in the details pane.

Package Type

A specific category of package, such as a Content Type or a Community. A package type always includes the same types of dependencies.

Packages View

When you click a packages node in the Server Groups Tree, the Multi-Server Manager displays the packages view in the details pane. For each package installed on the server, packages view displays: package name, package type, description, archive name, installation date, source server, Rhythmyx version, Rhythmyx build.

Publishing Lock

Lock Rhythmyx puts on the server when it begins installing an archive so you cannot publish content until archive installation is complete.

Server Dependencies

Elements that are not installed with Rhythmyx but must be present on the target server for a package to function correctly. They cannot be installed by the Multi-Server Manager.

Server Groups Tree

Panel on Multi-Server Manager that displays all of the servers registered with your Multi-Server Manager, the servers' repositories, and the descriptors, archives and packages they contain.

Shared Dependencies

Installable dependencies that are installed with Rhythmyx, must be present for an element to function properly, but are not a required part of its package. They are "shared" because they may be dependencies of other packages.

Source Server

The Rhythmyx Server containing elements that the Multi-Server Manager will transfer to another server.

System Dependencies

Elements that are installed with Rhythmyx and must be present on the target server for a package to function correctly, but cannot be installed by the Multi-Server Manager. System dependencies usually begin with the prefix sys_.

Target Server

The Rhythmyx Server on which the Multi-Server Manager installs Rhythmyx elements from another server.

Transaction Log

The transaction log is accessible from the Archive Summary dialog or the package log. For each element installed for each package, the Transaction Log tab displays the element's package, the element's dependency, the element name, the element type, and the installation action (created, modified, deleted, skipped, failed).

Transforms

Mappings for datasources or mappings for elements.

Typical Create Mode

Mode of the Create Archive/Descriptor wizard that only takes you to the Identify ID Types dialog if hard-coded identifiers in server objects that you are installing are unidentified.

Typical Installation Mode

Mode of the Install Archive wizard that only takes you to the Transforms (mapping) dialog if your datasource mappings or element mappings are not complete.

User Defined Dependencies

Files used by elements in a package that the Multi-Server Manager does not discover, but the user specifies.

Validations

Process completed by Install Archive wizard to see if there are any potential problems with elements in an archive that is about to be installed. Depending on the severity of the problem, it is reported as an error or a warning. For example, a problem could be that an element already exists on the target server or that a necessary dependency is not present in the archive. Users cannot install packages with errors, but can choose whether or not to install packages with warnings.
