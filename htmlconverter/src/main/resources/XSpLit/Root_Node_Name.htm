Root Node Name

Since XML field name syntax requires all fields in a document to share a common root node name, XSpLit automatically creates a root node name for all fields in an HTML page. It gets the root node name from the name of the associated DTD for the fields. This is the name of the HTML file that was split.

For example, an HTML page named person.htm includes the following code:

<span id="psx-name/first"></span> <span id="psx-name/last"></span>

<span id="psx-title"></span>

XSpLit gives the field psx-name/first the full name person/name/first, and psx-title becomes person/title. The associated DTD takes the name person.dtd. XSpLit does not modify the original HTML file.

See XML fields and DTDs for more information.

Note: XSpLit does not change field names in the HTML page that already use the correct root node name.
