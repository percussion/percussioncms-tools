Pre-selected and Pre-checked Form Fields

Form fields such as check boxes, radio buttons and selections often need to be pre-checked or pre-selected based on back-end data. This can be accomplished by setting the checked or selected attribute for the corresponding form field. For example:

<input type="radio" name="favorite" value="1" checked="psx-favorites/checked">English

<input type="checkbox" name="known" value="1" checked="psx-known/checked">English

<select name="magazines" size=4 multiple="multiple" selected="psx-magazines/selected">

<option value="0">None</option>

<option value="1">Time</option>

<option value="2">Newsweek</option>

<option value="3">People</option>

All dynamic tags must have the same parent except the dynamic tag for the pre-checked or pre-selected attribute. If the pre-checked or pre-selected dynamic tags use a different parent, the value attribute is used as the reference.

For example, the following HTML code defines checkboxes in the middle column of the table. Some of these boxes may be checked.

<table border="1" width="60%">
<tr bgcolor="lightgrey">
<th>Product</th>
<th>Select Your Favorite Features</th>
<th>Comment</th>
</tr>
<tr>
<td>psx-product/name</td>
<td>
<form method="POST">
<table><tr><td>
<input type="checkbox" name="C1" value="psx-product/feature/value" checked="psx-product/checked">psx-product/feature
</td></tr></table>
</form>
</td>
<td>
<form method="POST">
<textarea rows="2" name="S1" cols="20">psx-product/comment</textarea>
</form>
</td>
</tr>
</table>

The section of code within the <form> tags defines the column with the check boxes:

<form method="POST">
<table><tr><td>
<input type="checkbox" name="C1" value="psx-product/feature/value" checked="psx-product/checked">psx-product/feature
</td></tr></table>
</form>
