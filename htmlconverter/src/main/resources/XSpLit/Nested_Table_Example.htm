Nested Table Example

Suppose you had the fields psx-product/name and psx-product/feature in the following HTML code:

<table border="1" width="50%">
<tr>
<td><b>psx-product/name</b><br />Take a look at all these features!</td>
<td>
<table border="1" width="100%">
<tr>
<td>psx-product/feature</td>
</tr>
</table>
</td>
</tr>
</table>

XSpLit processes the outermost table first. The outermost table includes two fields, both sharing the node 'product.' This outer row table repeats once per product. The next table inward is the one nested inside the second column. The outermost table repeat has already exhausted the node 'product.' Therefore, this nested table row only repeats once per 'feature.'

At run time, one product with three features would produce the following output:

Content: XML

Output: HTML

<products>
<product>
<name>Widget</name>
<feature>It's cheap!</feature>
<feature>It's easy!</feature>
<feature>It's fun!</feature>
</product>
</products>

Three occurrences of product, each with varying occurrences of feature would produce this output:

Content: XML

Output: HTML

<products>
<product>
<name>Widget</name>
<feature>It's cheap!</feature>
</product>
<product>
<name>Gadget</name>
<feature>It's easy!</feature>
<feature>It's fun!</feature>
</product>
<product>
<name>Gizmo</name>
<feature>Low maintenance</feature>
</product>
</products>
