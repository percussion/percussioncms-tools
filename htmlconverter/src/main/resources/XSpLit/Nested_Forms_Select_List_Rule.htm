Nested Forms Select List Rule

About the Root Node

HTML does not support nesting Tables and Lists in the form Option tag. You can nest Select lists or an entire Form within a Table cell or List item. See Extended Table Nesting Rule or Extended List Nesting Rule for more information on how a repeating Select list affects repeat behavior when nested inside other repeating items.

Suppose you had the field psx-product/feature within the following HTML code:

<table border="1" width="40%">
<tr bgcolor="lightgrey" >
<th align="left">Product</th>
<th align="left">Features</th>
</tr>
<tr>
<td>psx-product/name</td>
<td>
<form method="POST">
<p>
<select name="choices" size="1">
<option selected value="none">Select a feature</option>
<option value="psx-product/feature">psx-product/feature</option>
</select>
<input type="submit" value="Go" name="Go">
</p>
</td>
</tr>
</table>

XSpLit processes the outermost repeating element, the table row, first. The row repeats once per product. The second column contains the nested Select list, which XSpLit processes next. .The outer table row repeat exhausted the product node, so an Option element repeats once for each feature.
