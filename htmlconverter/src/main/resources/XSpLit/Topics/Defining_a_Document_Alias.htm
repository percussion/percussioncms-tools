Defining a Document Alias

Use the psx-docalias tag to define a document alias. This tag requires the psx-docref attribute, which specifies the content of the alias. The format of the tag is

<!--psx-docalias="aliasname" psx-docref="URL"-->

Use aliasname when marking up the source code, wherever you want to call dynamic content contexts. You will need to define a different alias for each unique content context you want to call. The URL attribute points to the actual XML file that is merged with the XSL stylesheet to form the final Web page.

You can use an alias to specify either a static content reference or a dynamic content reference. To specify a static content reference, make the value of URL a static URL. For example:

<!--psx-docalias="logo" psx-docref="logo.xml"

This reference will include the contents of the URL logo.xml as one of the contents of the HTML page.

To specify a dynamic content reference, make the value of URL a dynamic field. For example:

<!--psx-docalias="departmentlogo" psx-docref="psx-department">

The element department from the main context will store the URL of the content to use for this context.  Dynamic content references must be in the main context or Rhythmyx will not be able to resolve them correctly.
