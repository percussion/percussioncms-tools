What is SPAN?

The word span can appear in two ways in HTML:

A td element can have colspan and rowspan attributes.  These attributes determine how many adjacent sells in a table use the particular attributes and style settings of the td element. This use of span is not of particular interest when using XSpLit.

XML also uses span as a markup tag, used to give structure and context to inline content. This use of span is important when using XSpLit. Use span to enclose a specific text block and apply any type of attribute on the text block , without adding formatting. The span element has no meaning on its own; use it with CLASS or ID attributes. You can nest span in all other tags, and nest all other tags within span. In many cases, span is recommended and in other cases, it is required.

CAUTION:  When formatting HTML 4.0, it is common to use span to apply cascading style sheet (CSS) formatting to elements enclosed in the span tag.  This practice usually works well because browsers do not hold HTML to strict standards of well-formedness.  Well-formed HTML, however, will not allow this practice.  XSpLit uses HTML Tidy to ensure that HTML is well formed.  HTML Tidy collapses span elements to one line if they contain an ID attribute.  Span elements that contain other elements are deleted.  The result is that any CSS formatting you meant to apply to your output HTML with the span element will be lost.  As an alternative, consider using the div tag or assigning the class attribute to other enclosing elements.
