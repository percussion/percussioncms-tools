Bindings Tab

The Bindings tab of the Template editor allows you to specify the variables you will use when processing your Templates, and to define the value for each variable.

Screenshot of Template Editor Bindings tab

Both Variables and Values are expressed using Java Expression Language (JEXL).

In the Variables table, the Variable Name column lists the variables used in the Template. You can choose a system variable or define a custom variable of your own.

Custom variables must begin with the character "$", but can otherwise be any string; for example, $name or $length. A single variable can take a list of values as a one-dimensional array. For example, you can assign a value to $name[0] ($name[0]=Brief), which creates a single element list. If you assign another value to $name[4] (for example, $name[4]=Category), you have created an array of five elements for the variable $name:

$name[0]=Brief

$name[1]=<empty>

$name[2]=<empty>

$name[3]=<empty>

$name[4]=Category

You can also define more complex variables by concatenating two strings with a dot. For example, if you defined the variables $circle.diameter and $circle.area, you have defined the variable $circle, which contains two additional variables, diameter and area. Each of the contained variables must exist in the data being processed, but their values can be empty. If a variable does not exist, processing of the binding fails, which causes processing of the Template to fail and ultimately causes the processing of any page using that Template to fail.

The Value column defines the value that will be bound to each variable listed in the Variable name column. Values are JEXL expressions, which can return any kind of Java object. When defining bindings for Velocity templates, you should define your expressions to return the following data types:

Data Category

Recommended Java Data Types

Numbers

java.lang.Short, java.lang.Integer, java.lang.Long, java.lang..Double

Dates

java.util.Date (Dates must be formatted)

Strings

java.lang.String

The Value is displayed in the Value column of the table, but you use the Expression Editor to create the Value. After clicking in the Value column, click in the When you click in a cell of the Value column, Rhythmyx puts the cursor in the Expression Editor, where you can define the Value.

The Expression Editor includes and autocomplete function. As you enter text, the Rhythmyx Workbench displays a list of existing binding variables and functions that match the text you have entered. You can select the variable or function you want to use.
