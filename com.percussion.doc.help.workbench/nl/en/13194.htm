Maintaining Variants

Variants are a Rhythmyx feature from Versions earlier than Version 6.0. A Variant defines how to produce a formatted output of a Content Item. It defines transformations and formatting rules that are applied to a Content Item, as well as the rules that define how each Variant interacts with other Variants when aggregating Content Items together. Each Variant is associated with a specific Content Type (although a Content Type may have any number of associated Variants). The specific formatting for each Variant is provided by one or more XSL stylesheets.

In Rhythmyx 6.0, Templates supersede Variants. Existing Variants will continue to work, and the Variant dialogs of the Workbench provide the functionality to maintain existing Variants. You can also add new Variants, but it is recommended that you use Templates for new implementations in Rhythmyx Version 6.0. Templates are generally easier to work with than Variants, and work directly with the new Assembly and Content Managers, while Variants still require an Assembly application and resource.

For detailed information about implementing Variants, see the Rhythmyx Technical Reference.

Use the New Variant Wizard to create a new Variant.

Use the Variant editor to maintain an existing Variant.

You can

  • create a new Variant;
  • modify an existing Variant; or
  • delete a Variant.
