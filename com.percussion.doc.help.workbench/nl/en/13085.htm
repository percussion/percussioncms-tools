Adding a Field Visibility Rule

Add a visibility rule when you are defining a field in a Content Type or when you are defining a shared field.

For a graphic of the Field Visibility editor and definitions of the fields discussed below, see Field Visibility editor.

To add a visibility rule:

  1. In the Field Properties Editor, click [Visibility].

    The Visibility Rule editor opens.

  2. In the Rule Details box, in Rule type, choose Conditionals or Extension.
  3. If you choose Conditionals, manually enter conditional expressions in the table.
    1. In Variable, enter the name of a field.
    2. In Op, enter a conditional operator such as < or =.
    3. In Value, enter a value.
    4. If you will enter another conditional expression in the next row, in Boolean, enter AND or OR to show the relationship between this expression and the next one.
    5. Repeat steps a through d for each conditional expression that you include.
  4. If you choose Extension:
    1. In the Name drop list, choose an extension.

      If parameters are required, the Parameters dialog opens automatically.

    2. Add parameters if required.
  5. Click [Add] to add the Rule to the Visibility table.
  6. If you will enter another Rule in the next row, in Boolean, enter AND or OR to show the relationship between this expression and the next one.
  7. Repeat steps 2 through 6 to add additional visibility rules for the field.
  8. Click [OK].

    The Visibility Rule is applied to the field and the Field Visibility editor closes.

Now you can:

  • In the Field Properties editor, configure additional properties.
  • In the Field Properties editor, click [Transform] to create field transformations.
  • In the Field Properties editor, click [Validation] to create field validation rules.
