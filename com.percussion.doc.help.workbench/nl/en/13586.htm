CMS Navigator and other Rhythmyx Views

The default Rhythmyx perspective includes the CMS Navigator, which is composed of the five main views that represent the conceptual model of a Rhythmyx system. The CMS Navigator views and the objects they display are:

  • Content Design View - Content Types, Keywords, Shared Fields, Localization elements, Content Editor system field definition object
  • Assembly Design View - Slots, Templates, Global Templates, Item Filters
  • UI Elements Design View - Content Editor UI elements, Content Explorer UI elements
  • Security Design View - Roles, Communities
  • System Design View - CMS Files, Extensions, Configuration files, Relationship Types, Sites, Workflows

In general, in CMS Navigator views, categories for objects appear in the order in which they are most frequently accessed, and objects within the categories are arranged in alphabetical order.

You can drag and drop (or copy and paste) some of the objects within CMS Navigator views. You cannot drag and drop (or copy and paste) objects across the views.

The Rhythmyx Workbench includes several additional views:

  • Community Visibility View - Lists the system's Communities and the objects visible to members of each Community.
  • Object Sorter - Lists a group of objects in another view and sorts them by a property that the user chooses. This view helps users find objects embedded in hierarchies. It displays all objects at the same level.
  • File Explorer View - Lists files accessible to the local system in Windows Explorer format.
  • Browser View - A browser window.

For users who are upgrading from a version of Rhythmyx prior to 6.0, the following views are also visible in the default perspective. New users can add them to their Rhythmyx perspectives:

  • Database Explorer - Lists databases available to the system.
  • XML Server View - Lists the pre-6.0 object categories Applications, XSL Variants, Extensions, and Configurations.
