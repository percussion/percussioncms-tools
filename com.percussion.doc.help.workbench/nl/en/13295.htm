Data Pipe

The Data Pipe defines the connection between the Repository and the DTD used to map Repository data to XML documents.

To be valid, a Data Pipe always includes the following objects:

  • Page Data Tank (Page Data Tank, XML Application Procedures)
  • Backend Data Tank (Creating a New Application, Backend Datatank)

In query resources, the Data Pipe will also include a and may include a Results Pager.

In update resources, the Data Pipe will also include Updater Properties and may include a Transaction Manager.

Right-click on the pipe and from the popup menu choose Properties to access the Pipe Properties dialog. This dialog allows you to veiw the default properties of the Data Pipe and add a description.
