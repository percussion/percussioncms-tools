Content Type Editor - Properties Tab

The last tab of the Content Type Editor is the Properties tab. It allows you to enable or disable searching for the Content Type, set up item transform and validation rules, configure pre- and post-processing extensions, and edit some of the fields you entered in the New Content Type Wizard.

Use this tab to:

  • Add extensions to the Content Type
  • Edit the Label, Description, and Workflows of the Content Type

    Screenshot of Content Type Editor, Properties Tab

Field Definitions

Content Type name - Read only. System name of Content Type.

Label - Name displayed to users for Content Type.

Description - Description of Content Type.

Allowed workflows - Lists all available Workflows for Content Type. Allowed Workflows are checked.

[All] - Checks all workflows.

[None] - Unchecks all workflows.

Default workflow - The Workflow assigned to a Content Item of the Content Type if no other Workflow is chosen. When the Default workflow is selected, it is automatically checked under Allowed workflows if it is not already checked. If the user unchecks the Default workflow under Allowed Workflows, the first checked Workflow becomes the Default Workflow. The drop list for this field includes all available Workflows for the Content Type.

Enable searching for this Content Type - When checked indexes content of the Content Type for searching. Default is checked.

Item Transforms and Validations - Allow defining of Item Input Transforms, Item Output Transforms, Item Validation Rules, Pre-Processing, and Post-Processing for the Content Type. The Input Transforms tab is displayed in the graphic above, but all five tabs are nearly identical.

Input Transforms tab - Lists extensions selected which change the format or value of Content Editor data before it is entered into the Rhythmyx repository. A common item transformation involves reformatting a date field. Note: In versions of Rhythmyx prior to 6.0, item transforms were referred to as item translations.

Output Transforms tab - Lists Extensions selected which change the format or value of Content Item data when it is retrieved from the Rhythmyx repository and listed in a Content Editor or page. A common output transformation involves spelling out an abbreviated word. Note: In versions of Rhythmyx prior to 6.0, item transforms were referred to as item translations.

Validation tab - Lists Extensions selected for checking the format or value of data entered in to the Content Editor when it is saved and entered into the Rhythmyx repository. An item validation rule could specify that a start date must precede an end date.

Pre-Processing tab - Lists Extensions selected for processing information in the Content Editor prior to the request to the Repository.

Post-Processing tab - Lists Extensions selected for processing information in the Content Editor following the request to the Repository.

Fields on these tabs:

Extension Displays selected Extensions. Extensions execute in the order in which they are listed. Clicking in a row accesses a drop list that includes all available extensions for the tab.

C - Opens a dialog for setting conditions or defining extensions. The conditions or extensions specify when the extension is executed. Appears on Input Transforms, Output Transforms, and Validation tabs only.

Item Transforms and Validations Description - Displays a description of an extension selected in the Extension table.

[^] - Moves the selected extension higher in the list.

[V] - Moves the selected extension lower in the list.

[+] - Adds a row for inserting an extension above the selected extension.

[X] - Deletes the selected extension from the table.

Maximum Number of Errors Before Stopping Indicates the number of validation errors that Rhythmyx can find in the Content Editor fields before stopping processing. Appears on Validation tab only.
