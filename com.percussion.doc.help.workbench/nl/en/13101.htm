CMS Navigator - UI Elements Design

The UI Elements Design view of CMS Navigator displays User Interface elements objects and their sub-categories.

The main objects in the CMS Navigator UI Elements Design view are Content Editor and Content Explorer. Below, descriptions of these objects and their children follow the graphical representation of the view.

Screenshot of UI Elements Design View

Content Editor Folder

The Content Editor folder includes two sub nodes:

  • System Controls - Opening the System Controls node opens the sys_templates.xsl file in a text editor in read-only mode. The sys_templates.xsl file defines all of the Content Editor controls that Rhythmyx supplies by default.
  • User Controls - Opening the User Controls node opens the rx_templates.xsl file in a text editor in edit mode. Use the rx_templates.xsl file to create custom controls.

Content Explorer Folder

The Content Explorer folder includes folders representing categories of Content Explorer user interface features:

  • Menus folder - Contains two categories of Menus:
    • System - Includes system Content Explorer menus. You can view but not edit these menus.
    • User - Includes user-defined Content Explorer menus. You can create, edit, and view these menus.

      System and User Menus include the following sub-folders. All menu entries appear in one of the sub-folders:

      • Cascading - A menu with entries and submenus built by an implementer or included with the Rhythmyx system. Cascading menus may include references to menu entries below them. The Menu Entries are listed under the following Menu Entry folder. To add a Menu Entry to a Cascading Menu, drag it from one of the Menu Entry folders and drop it on the Menu node.
      • Dynamic - A menu whose entries are generated when the user accesses it. Dynamic menus cannot include references to menu entries.
  • Menu Entries folder - Contains the categories System and User. System lists Rhythmyx-provided menu entries (menu choices) and User lists user-defined menu entries.
  • Display Formats folder - The Display Formats node lists all of the Display Formats defined for the system.
  • Searches - Includes two categories of searches:
    • Standard - Lists search queries configured using search criteria provided by the system's search engine.
    • Custom - Lists urls of custom searches configured in files and stored in the Rhythmyx root.
  • Views - There are two types of views:
    • Standard - Lists view queries defined in the Rhythmyx view dialogs.
    • Custom - Lists urls of custom view queries configured in files and stored in the Rhythmyx root.
