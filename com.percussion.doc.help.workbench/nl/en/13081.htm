Adding a Field Validation

Add a field validation when you are defining a field.

For a graphic of the Field Validation editor and definitions of the fields discussed below, see Field Validation Editor.

To add a field validation:

  1. In the Field Properties editor, click [Validation]

    OR

    in the Fields and Field Sets Editor, select a field and click [Validation].

    The Field Validation editor opens.

    NOTE: Overriding validation rules on shared or system fields in Content Types is not a valid action. If you attempt to override validation rules for shared or system fields used in Content Types a warning appears and an error may occur when you try to save a Content Type.

  2. To create conditions that determine whether or not the field validation runs, click [Prerequisites].

    A specialized version of the Rule Editor opens with a checkbox labelled Skip validation if field is empty.

    1. If you do not want the field validated unless it is filled in, check Skip validation if field is empty.
    2. Enter any other conditions or extensions in the Rule Editor.
  3. In the Rule Details box, in the Validation type drop list choose a validation type.

    Depending on the validation type that you choose, fields for entering parameters may appear below the drop list.

  4. Enter values for the parameters. See Default Validations and their Parameters for a list of the default validation types and descriptions of their parameters.
  5. Click [Add] to add the validation to the first available row in the Rule table.
  6. To show the relationship between this validation and the next one, choose AND or OR in the Boolean drop list.
  7. Repeat steps 3 through 6 to add additional validations for the field. Validations will execute in the order they are listed in the Rule table.
  8. To change the order in which a validation is executed on a field, select the validation in the Rule table and click Ù or Ú.
  9. To insert a validation between two existing validations in the Rule table:
    1. Select the lower validation and click +.

      An empty row appears between the two existing validations.

    2. Follow steps 3 through 6 to add a validation in the empty row.
  10. In Validation failure message, optionally enter a message to appear in the field if the validation fails.
  11. Click [OK].

    The validations entered in the tables are applied to the field, and the Field Validation editor closes. The Field Properties editor or the Field Set tab of the Content Editor reopens.

Now you can:

  • From a Fields and Field Sets editor, click [Properties] to go to the Field Properties editor.
  • From the Field Properties editor, configure additional properties.
  • From the Field Properties editor, click [Visibility] to create visibility rules.
  • From the Field Properties editor, click [Transform] to create field transformations.
