Rule Editor

Use the Rule Editor to specify the conditions that trigger an extension or another process.

You access the Rule Editor differently depending on where you are accessing it from. For example to access it from the Relationship Effects table, double click on the icon in the C column on table for the effect that you are entering. To access it from the Field Validation editor, click the [Prerequisites] button.

To enter a rule for performing an extension or other type of processing:

  1. Open the Rule Editor.

  2. In the Type column, choose Conditional or Extension.
    1. If you choose Conditional, double click in the Rule column and click [...] to open the Conditional Property dialog. Enter a condition and click [OK]. The Conditional Property dialog closes and the condition is entered in the Rule column of the Rule Editor.
    2. If you choose Extension, double click in the Rule column to open a drop list of extensions. Choose an extension. If the extension requires parameters, the Function Properties dialog automatically opens. Enter properties for the extension's parameters and click [OK]. The Function Properties dialog closes. The extension and any parameter values are entered in the Rule column of the Rule Editor. Note: All of the available extensions result in a boolean value (either TRUE or FALSE).
  3. In the Op column, enter AND or OR to show the connection to the next rule in the list. If there are no other rules, leave the value as null.
  4. Enter any number of rules.
