Slot Editor

Use the Slot editor to

  • complete a Slot definition; or
  • to maintain the data for a Slot.

The Rhythmyx Workbench displays the Slot editor when you click the [Finish] button after completing the Slot wizard. Other ways to access the Slot editor include:

  • In Assembly View, right-click the Slot you want modify and choose Open.
  • In Assembly View, double-click the Slot you want to modify.

Field Descriptions

Slot name Read-only. Displays the name of the Slot defined on the Slot wizard.

Label Label of the Slot displayed in Content Explorer.

Description Free-form description of the Slot.

Type Specifies the type of Slot. Options are Regular (Standard Slot; can be assigned to a Template or Variant) and Inline (Slot is used to add inline links and content to rich text fields; can be assigned to the Inline properties of a Content Editor, but cannot be assigned to a Template or Variant).

Allowed relationship Types Specifies the Relationship Type of the Relationship allowed in the field. Options include all Relationship Types in the Active Assembly Relationship Category. By default, this category includes the Active Assembly and Active Assembly-Mandatory Relationship.

Content finder Extension used to populate the Slot if the Slot is used on a Template (if the Slot is used on an XSL Variant, the Variant determines how the Slot will be populated). Options include:

None: Defaults to the Regular Content Finder.

Regular: populates the Slot with Content Items manually assigned by the user. Can take the following parameter:

  • template: specifies the Template to use when formatting the Content Items in the Slot. You can specify either the Name of the Template, or its ID.

Automatic: populates the Slot with a list of Content Items generated by a query. Takes the following parameters:

query

(required) defines the query string used to determine the list of Content Items for the Slot. The queries must use JSR-170 syntax. For details about JSR-170 syntax, see the JSR 170 page.

type

(optional) specifies the type of the query. In Rhythmyx Version 6.0, the only supported option is sql.

template

(required) specifies the Template to use when formatting the Content Items in the Slot. You can specify either the Name of the Template, or its ID.

max_results

(optional) specifies the maximum number of Content Items to return for the Slot. Defaults to unlimited if not specified.

Legacy Automatic: populates the Slot with a list of Content Items generated by a Rhythmyx XML query application. Takes the following parameters:

resource

path (relative to the Rhythmyx installation directory) to the XML query resource that generates the list of Content Items. The query resource must be based on the sys_AssemblerInfo DTD.

template

(required) specifies the Template to use when formatting the Content Items in the Slot. You can specify either the Name of the Template, or its ID.

Managed Navigation: populates the Slot with Managed Navigation nodes. Takes the following parameters:

node_to_return

(optional) Options are self (returns the navon for the current Content Item) or root (returns the navtree). If not specified, defaults to root.

template

(required) specifies the Template to use when formatting the Content Items in the Slot. You can specify either the Name of the Template, or its ID.

The browse button activates the Extension Editor, which you use to specify the value for each parameter of the Content finder.

Allowed Content table: Specify the Content Types that will be allowed in this Slot in the Content Type column. Options available in the drop list include all Content Types not already assigned to the Slot. For each Content Type you allow in the Slot, you must assign at least one Template. Options in the Template drop list include all Snippet Templates and XSL Variants for the Content Type that have not already been allowed for the Slot. You can allow multiple Templates for each Content Type, but each combination of Content Type and Template requires a separate row in the table.
