Defining Shared Field Folders

When you add or edit a shared field in a shared field set, you first access the Shared Field Defs folder in Content View. Beneath the Shared Field Defs folder one or more shared definition files are defined. Each shared definition file includes one or more shared field groups. The following graphic shows an example of this hierarchy:

[graphic]

To add or edit a shared field in the Field and Field Sets editor, you right-click on its shared field set folder and choose Open.

Rhythmyx provides you with a default shared field definition file that contains xxxx shared field groups. You can add shared field groups to the default shared field definition file or create shared field definition files that can contain shared field groups.

To create a user-defined Shared Field Definition file:

  1. In Content View, right-click on the Shared Field Defs folder.
  2. In the menu, choose Create Folder.

    A folder named Folder plus the next available number (Folder01, Folder02, and so on) appears under the selected folder:

    [graphic]

    Note that a shared field definition file that corresponds to the folder is created in Rhythmyx.

  3. Right-click on the new folder.
  4. In the drop menu, choose Rename.
  5. Type in a new name for the folder.

    The folder name becomes xxxxxx.

Now you can:

  • drag and drop a shared field set from another user-defined shared field definition file into this shared field definition file xxxx is this possible xxxxx.

To create a user-defined Shared Field Group:

  1. In Content View, right-click a shared definition file node.
  2. In the menu, choose Create Folder.

    A folder named Folder plus the next available number (Folder01, Folder02, and so on) appears under the selected node:

    [graphic]

    Note that a shared field group that corresponds to the node is defined in the associated shared definition file.

  3. Right-click on the new folder.
  4. In the drop menu, choose Rename.
  5. Type in a new name for the folder.

    The folder name becomes xxxxxx.

Now you can:

  • right-click on the shared field set node and add a field to the field set.
