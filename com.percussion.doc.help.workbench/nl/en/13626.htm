Editing Object Access

By default Roles and Communities have complete Design and Runtime access to system and user objects.

You can modify the access to any object shown in the Rhythmyx Workbench views.

For graphics and instructions on accessing the the dialogs, and definitions of the fields discussed below, see Object ACL dialog and Default Object ACL Settings.

To edit the access to an object:

  1. Open the Object ACL dialog or the Security dialog.
  2. Under ACL Entries, select the Role, Community, or user whose access to the object you want to change.
  3. Check and uncheck any of the permissions under Design access or runtime Access.
  4. To add an entry, click [New Entry].

    The Add ACL Entry dialog opens.

  5. For each Role or Community that you want to add to the ACL, choose the entry under Roles or Communities and click [>] to move it to the Add to ACL list.

    If you want to move all of the Roles or all of the Communities to the ACL click [>>] beside the Roles or Communities list.

    If you change your mind, select the choice under Add to ACL and click [<] beside Roles or Communities to move it back to the appropriate list.

    If you want to remove all of the Roles or all of the Communities from Add to ACL, click [<<] beside the appropriate list.

  6. To add a user to the Add to ACL list, click [Add User].

    The Add User dialog opens.

  7. Enter the name of the user that you want to add and click [OK].

    The Add User dialog closes and the user is added to the Add to ACL list.

    If you change your mind, select the user and click [Delete] to remove it from the Add to ACL list.

  8. Click [OK] to return to the Object ACL dialog.
  9. Set permissions for any new entries that you have added.
  10. Click [OK] to close the dialog and save your changes.
  11. If you want to delete an entry from the Object ACL dialog, select the entry and click [Delete].
  12. Click [OK].

    The Object ACL dialog closes and the permissions are set for the object.
