Configuring Additional Properties

To enter additional properties in a field defined in the Fields and Field Sets Editor or to access the editors for setting field validation, field visibility rules, and field transforms, open the Field Properties editor. The Field Properties editor is not available for a child field set selected in a Content Type editor's Content Type tab, but it is available for the fields in a child field set displayed on a Content Type editor's <field set> tab.

For a graphic of the Field Properties editor and definitions of the fields discussed below, see Field Properties Editor.

  1. To add additional properties to a field select it in the Field and Field Set editor and click [All Properties].

    The Field Properties editor opens. The Field Name, Data Type, Storage Size, Default Value, Label, Mnemonic, Control, Show in Preview and Allow this field to be Searched fields are filled in with values from the previous screen. If you are editing a field in a child field set, the editor enables the Show in Summary check box. Optionally, enter any of the following fields:

    1. Check the Treat data as binary check box if the field is a large data field and you want to avoid database errors.

      The Show 'Clear Field' Checkbox in Binary Control field automatically becomes enabled when you check Treat data as binary.

    2. In Error Label, enter a phrase to appear in the field if the user enters invalid data.
    3. For a field in a child field set, leave Show in Summary checked to display a field name and its values in the main Content Editor when users open it. Otherwise, users must click [Edit Table] to view the field as part of the entire child field set in a separate window.
    4. If Treat data as binary is checked, the Show 'Clear Field' Checkbox in Binary Control field is enabled. If you want the user to see a checkbox for clearing the data in the backend table column before inserting the data entered in this field, check the check box. If you do not make this field available, users may fail to overwrite data in the backend table with a null or blank value.
    5. If Allow this field to be searched is checked, additional options for searching are enabled:
      • To include formatting marks such as HTML tags in the search index for this field, check Index formatting, such as HTML tags.
      • To include this field in user searches by default, check Include in full text multi-field query. This field is checked by default.
      • To include punctuation that directly precedes or follows words as part of the words, check Punctuation is part of word.
    6. To add Visibility rules for the field, click [Visibility].
    7. To add Validation rules for the field, click [Validation].
    8. To add input or output Transforms for the field, click [Transforms].
    9. Click [OK] to save the field properties entered in this editor and return to the Field and Field Sets editor.

Now you can:

  • Continue to add fields in the Field and Field Sets editor
  • Define field Visibility rules
  • Define field Validation rules
  • Define field Transforms
