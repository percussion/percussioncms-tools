Extension Registration Editor for Java

The handler for an extension is defined as Java or JavaScript in the New Extension Registration wizard. If the handler is defined as Java, the following editor opens.

Screenshot of Extension Registration Editor for Java

Field Descriptions

Java extension name - Read only. Name of extension.

Description - Optional description of the extension.

Class name - Name of java class that defines the extension.

Context - Category of the extension.

Category - Category of extension defined for the Rhythmyx Workbench.

Parameters - This table lists the parameters that the extension uses.

Parameter Name - Name of parameter

Parameter Type - Data type of parameter

Parameter Description - Description of the parameter

Supported Interfaces - Supported interfaces define the functions that the extension can serve. For example, the com.percussion.extension.IPSUdfProcessor interface allows the extension to function as a UDF. Below the Supported Interfaces drop list is a box that displays the Supported Interfaces that you have selected.

Required files - Any supported or auxiliary required files needed to implement the extension. Required files are generally .class, .jar, or .zip files. Click [...] to browse for a required file. You can choose multiple required files. Below the Required files text box is a box that displays the Required files that you have selected.

Required applications - Applications that require the extension to run.Click [...] to browse for required applications. You can choose multiple required applications. Below the Required applications text box is a box that displays the Required applications that you have selected.

The purpose of this field is to enable the Multi-Server Manager to find the extension as a dependency of the application. Required applications do not effect the way the extension runs. For more information see the Rhythmyx Multi-Server Management document.
