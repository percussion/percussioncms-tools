Maintaining Relationship Types

In Rhythmyx, a Relationship is a logical association between two Rhythmyx objects. Examples of Rhythmyx functions that use Relationships are:

  • Active Assembly (the association between a Content Item and its related content)
  • Folders (the association between a folder and a Content Item contained in it)
  • Promotable Versions (the association between the original Content Item and the new Version)

Most commonly, the objects involved in the Relationship are Content Items, but Relationships are also used to implement the folder functionality in the Content Explorer user interface and can be extended to incorporate other Rhythmyx objects as well.

All Rhythmyx Relationships have the following properties:

  • Rhythmyx only recognizes one-to-one Relationships.
  • In each Relationship, one object owns the relationship (and is referred to as the owner). The other object is the dependent in the Relationship.
  • If an object is deleted, all of its Relationships are also deleted.

For more detailed information about relationships, see the document Implementing the Relationship Engine.

  • Use the New Relationship Type wizard to create a new Relationship.
  • Use the Relationship Type editor to set and maintain properties of Relationships.

You can:

  • Create a basic Relationship Type
  • Add properties to the Relationship Type
  • Edit properties of a Relationship Type
  • Delete Relationship Types
