Dragging and Dropping (Copying and Pasting)

You can drag and drop objects in views (or use cut/copy and paste in the right-click menus) to perform the following functions:

  • To create a reference to an object by dragging and dropping the object onto another object that can include references to it. Note: If a parent object of the object being dropped on is open in an editor, you cannot perform the drag and drop function. For example, if you drag a Menu Entry and drop it on a user-defined cascading Menu, a reference to the Menu Entry is created under the Menu.
  • To move an object by dragging and dropping it within the user-defined area of its parent object node. The user-defined area includes user-defined folders and the area directly under the parent object node if objects can be created outside of folders. For example, the user-defined area in the graphic below is encircled with a rounded rectangle.

  • To copy an object by dragging it from a sub-folder and dropping it on the main folder or node for the object (for example, in Security Design view dragging the Enterprise Investments Community on top of the Community Folder creates a copy of the Enterprise Investments Community.)

By default, the Rhythmyx views appear as tabs in a single window. To be able to drag and drop an object from one view to another, you must drag one of the views to another window.
