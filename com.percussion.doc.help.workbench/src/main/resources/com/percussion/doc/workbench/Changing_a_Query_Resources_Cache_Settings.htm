Changing a Query Resource's Cache Settings

Resource caching is an advanced feature, and you should consult with Rhythmyx Customer Support prior to changing any resource cache settings.

To change a resource's cache settings:

  1. In the Pipe Properties dialog, click [Cache Settings].

    Rhythmyx displays the Cache Settings dialog.

  2. To enable caching for the resource, check Enable Caching; to disable caching for the resource, uncheck Enable Caching.
  3. To add CGI variables, cookies, and user context variables to the default key for the cached page, click .

    Rhythmyx opens a Value Selector dialog that allows you to create CGI variables, cookies, and user context variables.

    Create as many additional key values as necessary.

  4. To delete a key value, select the value and click .
  5. To have Rhythmyx automatically flush the page if data obtained by a dependency changes, add the dependency manually by specifying the query resource(s) that obtain the data:

    Click the Dependencies tab.

    Click to open the Select Dependency dialog.

    In the Application Name drop list, choose the application name associated with the dependency, and in the Resource Name drop list, choose the resource used to add the additional data. Click [OK] to enter the dependency on the Dependencies tab.

    Add all dependencies that affect the result page data.

    To remove a dependency, select it and click .

  6. Click [OK] to save the cache settings.
