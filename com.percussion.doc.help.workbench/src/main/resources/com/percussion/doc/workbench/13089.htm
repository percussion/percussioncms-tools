Adding a Field Transform

Add a field transform when you are defining a field in a Content Type or when you are defining a shared field.

For a graphic of the Field Transform editor and definitions of the fields discussed below, see Field Transform Editor.

To add a field transform:

  1. In the Field Properties Editor, click [Transform].

    The Field Transform editor opens.

  2. Click the Input tab to define a field input transform; click the Output tab to define a field output transform.
  3. Click in the first available row in the Rule table.
  4. In the Rule Details box, in the Transform type drop list choose a transform type.

    Depending on the transform type that you choose, fields for entering parameters may appear below the drop list.

  5. Enter values for the parameters. See Default Transforms and their Parameters for a list of the default transform types and descriptions of their parameters.
  6. Click [Add] to add the transform to the first available row in the Rule table.
  7. Repeat steps 3 through 6 to add additional transforms for the field. Transforms execute in the order they are listed in the Rule table.
  8. To change the order in which a transform is executed on a field, select the transform in the Rule table and click or .
  9. To insert a transform between two existing transforms in the Rule table:
    1. Select the lower transform and click +.

      An empty row appears between the two existing transforms.

    2. Follow steps 3 through 6 to add a transform to in the empty row.
  10. In Transform failure message, optionally enter a message to appear in the field if the transform fails.
  11. Click [OK].

    The transforms entered in the Rule table are applied to the field, and the Field Transform editor closes. The Field Properties editor reopens.

Now you can:

  • Configure additional properties.
  • Click [Visibility] to create visibility rules.
  • Click [Validation] to create field validation rules.
