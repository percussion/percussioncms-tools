Cache Settings Dialog

Resource caching saves query result pages in the server cache to decrease page loading time. By default, some resources in Rhythmyx are set for caching and others are not, but you may change the cache settings. In general, only apply resource caching to queries based on data that does not frequently change.

NOTE: Resource caching is an advanced feature, and you should only use it when directed by Rhythmyx Technical Support.

Use the Cache Settings dialog to maintain cache settings. To access the Cache Settings dialog:

  1. Right-click on the Data Pipe and from the popup menu choose Properties.

    The Rhythmyx Workbench displays the Pipe Properties dialog

  2. On the Pipe Properties dialog, click the [Cache Settings] button.

Field Descriptions

Enable Caching If checked, caching is enabled for the resource.

Additional Key Settings By default, Rhythmyx identifies the cached page with a key using the URL's protocol, server, port, Rhythmyx root, page extension, and HTML parameters. However, CGI variables, cookies, and user context variables may be appended to the page and therefore, must be added to the key. Use this field to add these additional key variables.

Dependencies Specifies applications that will trigger an automatic cache flush if they change.
