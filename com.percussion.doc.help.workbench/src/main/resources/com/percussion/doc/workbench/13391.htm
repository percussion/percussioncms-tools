Maintaining Field Transforms

A field transform is a condition or extension that changes the format or content of data in a field. For example, a field transform could reformat a date entered into a specific field. The field transform is part of the field's definition; therefore, a shared field is affected by a field transform regardless of the Content Type that includes the field.

There are two types of field transforms:

Field input transform - Changes the format or content of a field before it is entered into the Rhythmyx Repository.
Example: A user enters the date September 25, 2005 into a Content Editor, and the field reformats the date as 20050925 before Rhythmyx enters it into the Repository.

Field output transform - Changes the format or content of a field after it is retrieved from the Rhythmyx Repository but before it is displayed in a Content Editor.
Example: The date 20050925 is retrieved from the Rhythmyx Repository and the field reformats the date as September 25, 2005 before displaying it in the Content Editor.

  • Use the Field Transform editor to add and maintain field transforms.

With the editor you can:

  • Add new field transforms;
  • Edit field transforms; or
  • Remove field transforms.
